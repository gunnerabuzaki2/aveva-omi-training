What Is External Content?

External content refers to media content that lives outside of a Galaxy but can be linked to a ViewApp and displayed at runtime. System Platform includes several pre-built viewer Apps that can render this content — web pages, Microsoft Word documents, images, PDFs, spreadsheets, and more. Rather than rebuilding everything from scratch, you simply point to existing media files and let the built-in viewers handle the display.

Key Concept: An external content item is a named reference created in the Graphics tab that identifies the location and type of external media. It can be linked to one or more layout panes or automation objects for display at runtime.

External content items are created within the Graphics tab and configured using the External Content Editor, which specifies the location of the media and the content type. These items are managed in the Graphics tab hierarchy, visible in both the System Platform IDE and the Toolbox or Assets areas of the Layout and ViewApp Editors.

To associate external content with a ViewApp, you can either manually assign the external content item to a layout pane, or link it to an automation object for auto-fill navigation. A single external content item can be linked to multiple objects, making it a flexible mechanism for displaying shared resources.

Windows Server Add Roles and Features Wizard — the Desktop Experience feature must be enabled on Windows Server 2012 to support video playback in ViewApps

Important Considerations for Media Files

When working with external media files, there are several important factors to keep in mind regarding connectivity, drive mapping, and operating system features.

Pitfall: If you are showing videos within a running ViewApp on Windows Server 2012, you must enable the Desktop Experience feature. This feature includes required media components that are not enabled by default on Windows Server 2012. On later operating systems, installing the Full GUI automatically includes Desktop Experience.

Here are the key considerations for your media files:

  • Remote media files: If you are placing media files on a remote computer in your network, ensure that the computer where the ViewApp runs has reliable connectivity to the remote storage location.
  • Mapped drive URIs: If you specify a media file location using a Windows mapped drive letter, the computer launching the ViewApp must have the same drive letter mapped to the same location. Different machines may have different mappings.
  • Video on Windows Server 2012: Enable the Desktop Experience feature through the Add Roles and Features Wizard, as shown above. Without it, video content will not render in the ViewApp.

Creating External Content Items

You create external content items in the Graphics tab of the System Platform IDE. Right-click the Galaxy name or a toolset name, then select New → External Content. The item appears with an identifying icon and a default name of ExternalContent_001, which you can rename at any time.

Creating a new external content item — right-click the Training toolset and select New → External Content
The External Content Editor — configure the URL or URI, Media Type, and Content Type(s) for the external content item
Before you configure: Before opening the External Content Editor, you must know three things: (1) the location of the external media, (2) the type of media (e.g., image, PDF, spreadsheet), and (3) the content types of the layout pane that will host the content.

The External Content Editor

Double-click an external content item in the Graphics tab to open it in the External Content Editor. The editor provides several fields for configuring how the external content is loaded and displayed.

URL or URI Field

The URL or URI field specifies the location of the external media. It can be set as a static string (Constant mode) or as a dynamic reference/expression. The Mode icon on the left of the field lets you switch between these modes.

URL/URI field configuration — the mode icon on the left lets you choose between Constant and Reference Or Expression modes

In Constant mode, an ellipsis button appears to the right that lets you browse for a file or specify an HTTP/HTTPS URL. In Reference Or Expression mode, the ellipsis opens the Galaxy Browser, where you can select references or build a custom expression to locate the media dynamically.

Other Editor Fields

FieldDescription
URL or URIThe location of external media. Can be a file path, HTTP URL, or dynamic expression.
OpenIn Constant mode, validates the media location and attempts to open it with the OS default application. Disabled in Expression/Reference mode.
Media TypeA two-part MIME identifier (e.g., image/png, text/plain) specifying the type of application needed to process the media. Available types appear in bold in the dropdown.
SuggestAuto-detects the media type based on the file extension. Only works in Constant mode.
Content Type(s)The content types assigned to this item. Must match the content types configured for the layout panes that will display the content at runtime.

Out-of-the-Box Viewer Apps

System Platform provides several built-in Apps for displaying external content. These viewers handle rendering and user interaction (zoom, search, paging) without allowing permanent edits to the source files.

Linking external content to an automation object — click the Link Content button (double-plus icon) in the Content pane on the Attributes tab

Linking External Content to Automation Objects

You can link an external content item to an automation object for auto-fill navigation. This means that when a user navigates to the object at runtime, the associated external content automatically appears in the layout. A single external content item can be linked to multiple objects.

To create this link, open the object in the Object Editor, go to the Content pane on the Attributes tab, and click the Link Content button. The Galaxy Browser opens, allowing you to navigate to and select the external content item.

Deleting External Content Items

To delete an external content item, right-click it in the Graphics tab and select Delete. A Confirm Delete dialog appears that lists any objects referencing the item, allowing you to review dependencies before confirming the deletion.

Confirm Delete dialog — review any references to the external content item before confirming deletion

Import and Export

External content items can be managed using standard import/export operations from the Graphics tab. A ViewApp containing Apps designed to display external content can be exported from one Galaxy and imported to another. The External Content Editor automatically recognizes new media types from imported Apps.

External content items support:
  • Export to and import from aaPKG files
  • Inclusion in Galaxy backups and restoration from backups
  • Bundling with layouts or ViewApps that reference them

Knowledge Check

Quiz

Q1: What is external content in the context of System Platform?

External content refers to media content outside of a Galaxy that can be linked to a ViewApp and displayed at runtime. This includes web pages, Word documents, images, PDFs, and spreadsheets.

Q2: What must you know before configuring an external content item in the External Content Editor?

You must know the location of the external media, the type of media (e.g., image, PDF), and the content types of the layout pane that will host the content.

Q3: What are the two modes for the URL or URI field in the External Content Editor?

Constant mode (for static file paths or URLs) and Reference Or Expression mode (for dynamic references or expressions using the Galaxy Browser).

Q4: Name three out-of-the-box viewer Apps for external content.

DocViewerApp (Word documents), ImageViewerApp (images), PDFViewerApp (PDFs), SpreadsheetViewerApp (Excel spreadsheets), and WebBrowser (web pages).

Q5: Can a single external content item be linked to multiple automation objects?

Yes. A single external content item can be linked to multiple objects for auto-fill navigation.

Q6: What special feature must be enabled on Windows Server 2012 to show videos in a ViewApp?

The Desktop Experience feature must be enabled through the Add Roles and Features Wizard. It includes required media features not enabled by default on Windows Server 2012.
