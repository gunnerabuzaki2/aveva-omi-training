Section 3 — Visualization Overview

Application Server provides tools for developing interactive and animated graphical displays. The behavior of these displays — including animations and visual representations — can be associated with properties of processes and equipment through real-time data provided by System Platform. System Platform's visualization clients provide multiple user interface capabilities, including keyboard, mouse, tablet, and touch-screen inputs, as well as multi-screen outputs.

Visualization Clients

System Platform clients for visualization are Operations Management Interface and InTouch for System Platform. Both run the operator interface and display data from objects defined in an Application Server Galaxy. They can coexist in the same System Platform solution and share data and graphics.

Operations Management Interface (OMI) leverages both HMI and SCADA systems to deliver a convergence platform for both Operational Technology (OT) and Information Technology (IT). It provides a visualization engine for rich, modern user experiences across platforms.

InTouch for System Platform provides traditional development tools for creating displays and corresponding navigation. It is not included with AVEVA System Platform Enterprise.

Visualization Content

Several types of content can be used to build visualization client applications. Understanding these content types is essential for designing effective operator interfaces.

Industrial Graphics allow you to create customized graphical representations of processes and integrate other visualization tools into your application. They can be embedded or associated with automation objects in Application Server, so that alarms, history, logic, graphics, and so on can all be defined with the same automation object. This enables greater flexibility through object reuse (objects derived from templates and preconfigured for their environment) and centralized development (graphical representations maintained in one place). Industrial graphics are also referred to as symbols in Application Server.

Client Controls provide functionality contained in .NET controls that you can use in symbols. External .NET controls can be imported into a Galaxy and used in symbols by embedding them. Symbols with embedded .NET controls are only supported by InTouch for System Platform — to use a .NET control in an OMI application, it must be imported as an OMI App.

Operations Management Interface Content

The following content types are designed and supported in OMI applications only:

Content TypeDescription
Screen ProfilesDetermine where — on which display screens — you want to present content in a running OMI application.
LayoutsDefine how you want to organize content in a running OMI application.
OMI AppsPrebuilt, special-purpose collections of one or more controls that can be added to layouts to display content. Sometimes called "Apps."
WidgetsMini-apps that can be added to OMI applications. Also supported for InTouch and Web Client. Generally display data, can be interactive and resizable. Sometimes called "HTML5 Widgets."
External ContentMedia content outside of a Galaxy that can be linked to an OMI application and displayed at runtime.

AVEVA Connect for Graphics

AVEVA Connect provides a cloud-based repository that lets you share graphics between locations, Galaxies, and nodes. You can sign in to AVEVA Connect within the System Platform IDE to add it to the Graphics toolbox, providing a centralized storage and access point for reusable industrial graphics.

Graphics in AVEVA Connect are stored in stores (global, tenant, or user-specific), and multiple drives can be configured within each store with different access levels. An AVEVA Connect account is required. Users can have read-only or read/write access depending on administrator grants, and multiple users can access graphics in the same drive — but only one user at a time can edit or save a graphic.

Section 4 — Encrypted Communication

The end-to-end communication between software applications (server and client) can be encrypted and secured to prevent eavesdropping and malicious tampering — also known as Man-in-the-Middle attacks. To enable encrypted network communication, one of the nodes in the network must be configured to host and run the System Management Server (SMS).

The System Management Server (SMS) generates, manages, and distributes secure digital certificates used for establishing and maintaining secure communications. All other nodes need to connect to the SMS so they all become part of the same community. The SMS does not need to be available at runtime — once a node has joined, it will not need access until it is time to renew certificates.

Protocols Secured by Encryption

With encrypted communication, the following protocols are secured:

ProtocolPurpose
SuiteLinkPrimary communication protocol for real-time data exchange between Application Server and clients
Message Exchange (MX)Messaging protocol used for inter-component communication
iDataData access protocol used by web clients
iBrowseBrowsing protocol for configuration and management
HCALHistorian Communication Abstraction Layer for historian data replication

Software Supporting Secured Communication

The following software supports secured communication (all require version 2017 Update 3 or later):

  • Application Server 2017 Update 3 and later
  • InTouch Operations Management Interface 2017 Update 3 and later
  • InTouch for System Platform 2017 Update 3 and later
  • OI Server (all versions — communications from any OI Server to any other listed product can be encrypted)
  • InTouch Web Client 2017 Update 3 and later
  • Historian 2017 Update 3 and later
  • Sentinel System Monitor 1.1 for System Platform 2017 Update 3 and later
Architecture diagram showing secured communication paths (marked with blue lock icons) between System Platform components: Historian replication via HCAL, Application Server to Historian via HCAL, OMI to Application Server via MX, InTouch connections via SuiteLink, web server access via iData, and System Monitor via HTTPS — all encrypted end-to-end.

Node-to-Node Communication

Node-to-node communication is unsecured under certain scenarios:

  • The software version on one or more nodes is earlier than 2017 Update 3 (Communication Driver version has no impact)
  • One node points to an SMS, but the other does not have an SMS configured
  • Neither node has an SMS configured
Critical: If one node points to one SMS and another node points to a different SMS, there will be NO network connection between them. All nodes must point to the same SMS to form a single secure community.
Three network configuration scenarios: (1) Community 1 in Secured Mode — all nodes point to SMS1 with green shield icons and green connection lines; (2) Community 2 in Secured Mode — all nodes point to SMS2 with green shields; (3) Legacy Unsecured Mode — no SMS or old software with yellow exclamation icons. Between communities: red X = "No connection." Between secured and legacy: yellow = "Unsecured connection."

Licensing

Two licensing models are available: Perpetual licensing offers permanent licenses with no expiration date (tied to a specific purchased version). Flex licensing offers subscription-based licenses renewed for a specific term, using credits redeemable for a wide range of software. Visit sw.aveva.com/flex-subscription for more information on the Flex program.

AVEVA software is license-enforced using an activated licensing framework with an activation code. Management and activation are performed with two components:

  • License Server: Acquires, stores, maintains, and serves licenses. Supports redundancy and failover.
  • License Manager: A web-based user interface to access and maintain licenses.

License activation can be performed online (Internet access on the License Server network) or offline (activation off-site, still requires Internet access in both cases).

System Platform Perpetual Licenses

Perpetual licenses are available in different sizes, each combining a certain number of I/O points, Historian tags, and Communication Driver licenses. Each license includes one AVEVA Historian Client Web license and Remote Response Objects (downloadable from the Support site).

Important notes on perpetual licenses:
  • One System Platform license is required per Galaxy.
  • Licenses with 25K I/O points or less do not include a Microsoft SQL Server Standard license.
  • Licenses are for runtime only — a Development Studio license is required for development (offered only with the subscription model).

Supervisory Client Licenses

A Supervisory Client license enables the use of both OMI and InTouch for System Platform. It can be used for thick clients (desktop), thin clients (remote access via Remote Desktop Server), or web clients through InTouch Access Anywhere. Licenses are available with or without a Historian Client license, and a read-only option is also available.

Each node running a supervisory client consumes one Supervisory Client license. As with perpetual licenses, a Development Studio license is required for development.

AVEVA Operations Control

AVEVA Operations Control is a commercial offering that includes access to the AVEVA portfolio of operations control software, licensed with AVEVA Flex subscription using a named-user model. It is available in three editions:

Azure Active Directory Integration

Starting with System Platform 2023, user identity authentication and single sign-on via Azure Active Directory (now Azure Entra ID) is supported. After setting Galaxy security to use an external identity provider, Azure AD can authenticate users. System Platform users can then use their Microsoft-managed identities to log into the System Platform IDE, Operations Control Management Console, OMI ViewApps, and to perform secured and verified writes.

🧠 Knowledge Check

Q1: What are the two main visualization clients for System Platform?

Operations Management Interface (OMI) — which provides a modern, rapid-design framework for OT/IT convergence — and InTouch for System Platform — which provides traditional development tools for creating displays and navigation from scratch.

Q2: What types of content are unique to OMI applications?

Screen profiles (where content appears), Layouts (how to organize content), OMI Apps (prebuilt special-purpose controls), Widgets (interactive mini-apps), and External content (media outside the Galaxy linked to OMI).

Q3: What is the role of the System Management Server (SMS) in encrypted communication?

The SMS generates, manages, and distributes secure digital certificates for establishing and maintaining encrypted communications. All nodes must connect to the same SMS to form a single secure community. The SMS is not required at runtime — only for initial certificate distribution and renewal.

Q4: What happens if two nodes point to different SMS instances?

There will be NO network connection between them. All nodes must point to the same SMS to form a single secure community. This is a critical configuration requirement.

Q5: What are the two licensing models available for System Platform?

Perpetual licensing (permanent licenses with no expiration, tied to a specific purchased version) and Flex licensing (subscription-based, renewed for a specific term, using credits redeemable for software access). Development requires a Development Studio license, which is offered only with the subscription model.
