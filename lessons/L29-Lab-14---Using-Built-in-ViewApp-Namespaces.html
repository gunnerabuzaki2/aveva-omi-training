🧪 L29: Lab 14 – Using Built-in ViewApp Namespaces

Module 7 — ViewApp Namespaces | Lab 14

Pages 397–408

Introduction

In this lab, you will create a symbol with two buttons for navigating to the previous and next sibling navigation items within the Line1 and Line2 areas at runtime. When configuring animations for the buttons, you will use attributes from the Navigation namespace to control whether the buttons are active or disabled at runtime, based on whether a next or previous sibling navigation item is available within the area.

Key Concept — ViewApp Predefined Namespaces: ViewApps provide built-in namespaces such as Navigation that expose runtime state about the current view. The Navigation namespace includes attributes like Current (current navigation item), PrevSibling/NextSibling (adjacent items), and PrevSiblingCount/NextSiblingCount (count of available siblings) — all of which can be used in animation expressions and references.

You will then add a pane to Asset_Layout and assign the symbol to it.

Objectives:
  • Use the attributes in the Navigation namespace
  • Add custom navigation buttons to a layout

Button Interface Preview

The SiblingNavigation symbol will contain two flat gray rectangular buttons — "Previous" on the left and "Next" on the right — arranged in a grid container.

Create a Sibling Navigation Symbol

Step 1: On the Graphics tab, in the Training \ Symbols toolset, create a symbol and name it SiblingNavigation.

Step 2: Double-click SiblingNavigation to open it in the Graphic Editor.

Step 3: Draw a button on the drawing canvas and enter Previous for the text. The button is auto-named Button1. In this example, the Appearance \ Width property is 100 and the Appearance \ Height property is 30.

The Graphic Editor showing a gridded white canvas with a light gray "Previous" button (Button1) selected, displaying blue resize handles on all edges and corners.

Step 4: Double-click Button1 to open the Edit Animations dialog box.

Step 11: On the Properties tab for Button2, modify the Appearance \ Text property to Next.

Button2 text updated to "Next" — two grid-aligned buttons: Previous (left) and Next (right, selected with blue resize handles).

Step 12: Right-click Button2 and select Substitute | Substitute References.

Step 13: Use Find & Replace to replace all occurrences of Prev with Next. Be sure to click Replace All. When finished, ensure the references in the New column are correct:

Step 17: With Pane1 selected, click the Add new pane up-arrow to add a pane above. A new pane named Pane4 is added above Pane1.

The Layout Editor for Asset_Layout showing the newly added Pane4 at the top, with Pane1 (default, selected with blue border), Pane3, and Pane2 (containing Asset Information and Device Information tabs) visible below.

Step 19: On the Toolbox tab, search for the SiblingNavigation symbol.

Toolbox tab with search string "sibli" entered, showing 1 matching content item — the SiblingNavigation widget with its Previous and Next button preview.

Step 20: Drag the SiblingNavigation symbol to Pane4.

The Layout Editor showing SiblingNavigation (Previous and Next buttons) placed in Pane4 at the top of the layout, above the main content panes.

Step 21: Click the pane divider between Pane1 and Pane4, but do not move the divider. Divider properties appear on the Properties tab.

Step 25: Navigate to Mixer100.

Notice that the Previous and Next navigation buttons appear on the top-right corner of the mixer graphic. The divider is not visible, which makes the navigation buttons appear to be part of the mixer graphic.

Knowledge Check

Q1: Which Navigation namespace attribute determines the current navigation item?

MyViewApp.Navigation.Current returns the current navigation item and is used as the reference for the Pushbutton animation's Set action.

Q2: How does the Disable animation know when to disable the Previous button?

The expression MyViewApp.Navigation.PrevSiblingCount == 0 evaluates to true when there is no previous sibling, which triggers the Disable animation to disable the button.

Q3: What is the purpose of the Element Style animation in this lab?

The Element Style animation visually distinguishes active and inactive buttons — when the sibling exists, it applies the "Navigation" element style (normal appearance); when no sibling exists, it applies "Intensity2" (dimmed/grayed appearance).

Q4: Why do we hide the pane divider at runtime?

Hiding the divider makes the navigation buttons appear to be seamlessly part of the mixer graphic, creating a more integrated and polished operator interface.
