What Are Widgets?

Widgets are "mini apps" that you can embed in your visualization applications. They are reusable controls built with web technology that generally display data and provide on-screen user interface elements integrating with other platforms and data sources. Unlike a single icon, widgets take up more space in the interface and can be interactive and resizable. They are supported in Operations Management Interface applications.

Widgets vs. OMI Apps: Widgets are self-contained programs that run automatically to show information when the ViewApp starts. OMI Apps, by contrast, are typically programs that you must interact with to run. This makes widgets ideal for dashboards, live data displays, and other always-on content.

Out-of-the-Box Widgets

Application Server provides several widgets out of the box, available from the Graphics tab in Galaxies created with the Default or Default_EMPTY Galaxy types. Each widget serves a distinct purpose and can be configured through its properties.

Carousel Widget

The Carousel widget cycles through a series of selected graphics, rotating them at a configurable, predefined interval. It is commonly used to display dashboards, alerts, or alarm information on large monitors on the plant floor. The carousel supports keyboard navigation, mouse-hover pausing, and continuous looping.

PropertyDescription
AutoplayIf True, the carousel automatically starts on load. If False, the user must select the next item. Default: True.
BackgroundColorThe background color of the widget.
GraphicNamesA comma-separated list of graphics for the carousel to display at runtime. Linked graphics that are renamed will appear as empty graphics.
IntervalThe delay in milliseconds between automatically cycling to the next graphic.
KeyboardIf True, the carousel responds to keyboard inputs. Default: True.
LoopIf True, the carousel continuously cycles. If False, it stops after a single cycle. Default: True.
PauseIf True, the carousel pauses on mouse-enter or touch-down and resumes on mouse-leave or touch-up. Default: True.

Datagrid Widget

The Datagrid widget displays tabular data within a layout pane during runtime. Users can filter the data displayed in the widget, and the widget includes visual properties to customize its appearance and the formatting of the tabular data.

MapApp Widget

The MapApp widget shows a map containing interactive symbols within a running application. During runtime, the map provides pan and zoom controls with touch support. Clicking a symbol that represents an asset can automatically navigate the ViewApp to that asset. Symbols on the map typically use latitude and longitude for accurate geo-location.

QRCode_Scanner Widget

The QRCode_Scanner widget connects to a device camera to scan a QR code and returns the result as a string. To use this widget, the device showing the ViewApp must have a camera — it is recommended for use on a physical machine rather than a virtual machine. When using the Web Client remotely, you must use a secure URL (https://).

PropertyDescription
QRCodeThe resulting string of the scanned QR code.
AutoStartIf True, the camera starts automatically on load.
AutoStopIf True, the camera stops after the QR code is scanned.
StartStopIf True, the camera starts. If False, the camera stops.
BackgroundColorThe background color of the widget.
Tip: You can configure a script to read the QR code and display a graphic based on the scanned value. At runtime, the QRCode_Scanner object shows Start Camera and Stop Camera buttons. If the scanned code matches a script return value or graphic name, the corresponding graphic displays.

Teamwork Widget

The Teamwork widget is a Software-as-a-Service (SaaS) interface that connects workers to AVEVA Teamwork. It supports task management, learning goals, issue identification and resolution, and standards documentation. Industrial organizations use Teamwork for skills development, knowledge sharing, and collaboration management across the enterprise from the cloud. The widget can provide training videos, a digital logbook, or a help-call interface.

Web_Browser Widget

The Web_Browser widget displays a web site within a layout pane.

PropertyDescription
URLThe address of a web page to display. Must be a valid URL. If the Web Client runs in HTTPS, only HTTPS pages can be loaded. If the site blocks cross-domain access, the widget will not work.

Assigning a Widget to a Layout Pane

For Operations Management Interface ViewApps, you can assign widgets to a pane from either the Layout Editor or the ViewApp Editor. Select the widget from the Toolbox tab, then drag and drop it from the preview area to the target pane. Once assigned, you can configure the widget's properties from the Properties tab.

Assignment steps:
  1. Select the widget from the Toolbox tab (Widgets toolset)
  2. Drag the widget to the target layout pane
  3. Configure properties via the Properties tab

Layout Considerations

When designing widget layouts, it is tempting to lay out widgets according to the dimensions of a specific device you develop with. While this can be a useful initial approximation, keep these important factors in mind:

  • Device variability: The number, size, and spacing of cells can vary widely from device to device. Your widget must be flexible enough to accommodate more or less space than anticipated.
  • Size buckets: As the user resizes a widget, the system responds with a device-independent (dp) size range. Planning your resizing strategy across "size buckets" rather than variable grid dimensions gives the most reliable results.
Important: Do not design widgets for a single fixed grid size. Always plan for multiple size buckets to ensure your widgets render correctly across different devices and screen resolutions.

Gestures

Because widgets live on the home screen, they must co-exist with the navigation established there. This limits the gesture support available compared to a full-screen App. For example, while Apps may support a view pager for lateral screen navigation, that gesture is already reserved on the home screen for navigating between home panels.

The only gestures available for widgets are touch tap/hold and vertical swipe. Horizontal swipe is reserved for home screen panel navigation.

Import and Export Widgets

You can import custom widgets into your Galaxy for use in OMI ViewApps or InTouch applications. From the Galaxy menu, select Import → HTML5 Widget and then select a .CWP (Custom Widget Package) file. The .CWP file bundles HTML5, CSS, and Javascript files together.

When the import is complete, the widget appears in the Galaxy Toolbox and is available for use in symbols or layouts. Widget properties can be configured after the widget is embedded in a symbol or assigned to a layout pane. Widgets can also be imported and exported as Galaxy objects using aaPKG files.

Widget import formats:
  • .CWP files (Custom Widget Packages) — bundle HTML5, CSS, and Javascript
  • .aaPKG files — standard Galaxy object export/import format

Knowledge Check

Quiz

Q1: What is the key difference between widgets and OMI Apps?

Widgets run automatically when the ViewApp starts to show information, whereas Apps are typically programs that require user interaction to run.

Q2: Name the six out-of-the-box widgets provided with Application Server.

Carousel, Datagrid, MapApp, QRCode_Scanner, Teamwork, and Web_Browser.

Q3: What does the Carousel widget's Pause property control?

If True, the carousel pauses when it detects a mouse-enter or touch-down event and resumes on mouse-leave or touch-up. If False, it will not pause. Default is True.

Q4: What file format is used to import custom widgets?

.CWP (Custom Widget Package) files, which bundle HTML5, CSS, and Javascript. Widgets can also be imported/exported as .aaPKG files.

Q5: Why is it important to design widgets for "size buckets" rather than fixed grid dimensions?

Because the number, size, and spacing of cells vary widely across devices. Planning for device-independent (dp) size ranges (size buckets) ensures widgets render correctly across different devices and screen resolutions.

Q6: What gestures are available for widgets on the home screen?

Only touch tap/hold and vertical swipe. Horizontal swipe is reserved for home screen panel navigation.
