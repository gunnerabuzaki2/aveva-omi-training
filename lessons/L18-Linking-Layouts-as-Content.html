📖 L18: Linking Layouts as Content

AVEVA™ Operations Management Interface 2023 — Module 4: ViewApp Application Customizations

Pages 289–290 (Section 1 – Link a Layout as Content in an Object, pp. 4-3 to 4-4)

Introduction

The link content feature allows you to associate a layout from the Graphics tab with an asset or template in the Galaxy. This is a powerful way to organize your ViewApp content — instead of manually placing graphics into every pane, you can link an entire layout to an automation object and let the system handle content display at runtime.

To link a layout, open the Object Editor for the target asset or template, navigate to the Attributes tab, and click the Link Content button in the Content pane. This opens the Galaxy Browser, where you can browse the Graphics tab and select a layout to link. A preview of the selected layout appears directly in the Object Editor so you can confirm your choice before committing.

Key Concept: Linked Layouts Must Pre-Exist
The layout you want to link must already exist in the Graphics tab. You cannot create a new layout from within the Object Editor — the Add button in the Content pane is for linking existing layouts only. If you need a new layout, create it first in the Graphics tab, then return to the Object Editor to link it.

Working with Linked Layouts

Once a layout is linked to an object, you can rename it and add a description directly in the Object Editor. The linked layout remains in the Graphics tab as a standalone item, but it is also associated with the automation object as content. This means the layout is accessible from both locations — you can open it for editing from the Graphics tab or from the Object Editor by double-clicking it or clicking the Edit button.

When editing layouts, a linked layout can also be assigned to a pane within another layout. At runtime, when a user navigates to an object that has a linked layout, that layout and its content appear in the running ViewApp. The layouts themselves are configured in the Layout Editor, including the content type for the layout and the content types for individual panes within it.

AutoFill with Linked Layouts

The AutoFill feature works hand-in-hand with linked layouts to automatically populate empty panes with relevant content. When AutoFill is enabled for a layout, any empty panes that have the Use For Autofill property enabled will attempt to fill with content from the linked asset using the Current Only AutoFill setting.

Key Concept: AutoFill Fills in Alphabetical Order
AutoFill populates empty panes based on the alphabetical order of layout names, symbol names, and other linked items. The naming convention of your layouts and symbols directly affects which content appears in which pane. Plan your naming strategy accordingly.

There are a couple of important limitations to keep in mind. When a layout is assigned to or displayed in another layout pane, the navigation buttons in that pane are not visible. If you need to navigate to other content from within an assigned layout, you will need to build custom symbols or scripts to handle that navigation.

⚠ Multi-Galaxy Limitation
In a multi-Galaxy environment, AutoFill is unable to fill panes if the OwningObject property for the linked layout is set to an object in a remote Galaxy. Ensure your layouts and their owning objects reside in the same Galaxy to avoid this issue.

Knowledge Check

Q1: Can you create a new layout from within the Object Editor's Content pane?

No. The layout must already exist in the Graphics tab. You can only link existing layouts — the Add button in the Content pane is for linking, not creating.

Q2: What order does AutoFill use to populate empty panes?

AutoFill fills panes in alphabetical order based on the names of layouts, symbols, and other items linked to the object.

Q3: What happens to navigation buttons when a layout is assigned to a pane in another layout?

The navigation buttons in that pane are not visible. You would need custom symbols or scripts to provide navigation functionality.

Q4: In a multi-Galaxy environment, what prevents AutoFill from working?

If the OwningObject property for the linked layout is set to an object in a remote Galaxy, AutoFill cannot fill the panes.
