📖 L15: Lab 7 – Creating a Custom Symbol

Source: AVEVA™ Operations Management Interface 2023 — Module 3: Industrial Graphics

Pages 237–258 (Book pages 3-99 to 3-120)

Introduction

In this lab, you will create a custom CommandPanel symbol to serve as a common command panel for opening and closing valves, starting and stopping pumps and agitators, and changing the speed setpoint for an agitator. The lab walks you through creating and configuring custom properties, adding button elements with Pushbutton, Element Style, and Value Display animations, embedding a speed setpoint entry symbol, and configuring Show Symbol animations so that clicking equipment symbols in runtime opens the appropriate command panel.

Objectives — Upon completion of this lab, you will be able to:
  • Create and configure custom properties in a symbol
  • Add Button elements and configure Pushbutton animations
  • Use Element Style Animations
  • Use Show Symbol animations
  • Configure the FaceplateMode property for a symbol
  • Configure the TreatAsIcon property for embedded symbols

Example Command Panels

The finished CommandPanel symbol adapts its display based on the type of equipment it controls. For a valve, it shows Open and Close buttons. For a pump, it shows Start and Stop buttons. For an agitator, it additionally shows a speed setpoint entry field. The title bar always displays the name of the device being controlled.

Valve command panel at runtime — clicking Inlet1_001 opens a pop-up panel titled "Inlet1_001" with Open and Close buttons. The panel appears at the mouse click position with its top-left corner aligned to where you clicked.
Agitator command panel at runtime — the Agitator_001 panel includes Start and Stop buttons plus a Speed setpoint entry field (unique to agitator objects that have the speed setpoint attribute).

Create the CommandPanel Symbol

Begin by creating a new symbol in the Training toolset and opening it in the Graphic Editor. You will then right-click the canvas to access the Custom Properties dialog, where you will define the properties that drive the panel's behavior.

Steps 1–4: On the Graphics tab, in the Training → Symbols toolset, create a symbol and rename it to CommandPanel. Double-click it to open in the Graphic Editor. Right-click the drawing canvas and select Custom Properties to open the Edit Custom Properties dialog. Click the Add custom property button (+) to begin creating the first property.

Edit Custom Properties dialog — initial empty state. The left pane shows "No Custom Properties are configured. Click '+' to add." Click the + button in the top-right to create the first custom property.
Close-up of the Add custom property button — the + icon in the top-right of the Custom Properties pane is used to add new properties.

Configure the Cmd Property

The Cmd property is a Boolean that sends a command to the device. Name the new property Cmd, set the Data Type to Boolean, delete the default value False and enter Me.Cmd, and add a description. The relative reference Me.Cmd will be resolved within context at runtime.

Steps 5–8: Name the property Cmd. Keep Data Type as Boolean. In Default Value, enter Me.Cmd. In Description, enter: "A Boolean value that sends a command to the device."

Cmd property configuration — Boolean data type with default value Me.Cmd, Public visibility, and description explaining it sends a command to the device.

Configure CmdOnMsg Property

The CmdOnMsg property is a String that holds the text displayed on the "on" button (e.g., "Start", "Open"). After adding the property and setting its Data Type to String, you must switch to Expression or Reference mode by clicking the toggle button to the left of the Default Value field. The icon changes to indicate expression mode is active, and the field gets a red outline. Then enter Me.Cmd.OnMsg.

Steps 9–13: Add property CmdOnMsg. Set Data Type to String. Click the mode toggle icon. Enter Me.Cmd.OnMsg as the Default Value. Add description: "A String value that can be used to display the actual True command message, such as Start, Open."

Switching to Expression/Reference mode — the toggle button to the left of the Default Value field changes the input mode from static text to expression evaluation.
CmdOnMsg property fully configured — String data type, expression reference Me.Cmd.OnMsg, Public visibility, with description text filled in.

Configure CmdOffMsg Property

The CmdOffMsg property follows the same pattern as CmdOnMsg but references the "off" message (e.g., "Stop", "Close").

Steps 14–19: Add property CmdOffMsg. Set Data Type to String. Switch to Expression/Reference mode. Enter Me.Cmd.OffMsg. Add description: "A String value that can be used to display the actual False command message, such as Stop, Close." Click OK to close the dialog.

All three custom properties configured — Cmd (Boolean), CmdOffMsg (String), and CmdOnMsg (String) are listed in the left pane with their respective default values.

Add Button Elements

With the custom properties in place, you now add visual elements to the symbol. Select the Button tool from the Tools pane and draw a button on the canvas labeled "Open". This creates Button1, which you will configure with three animations: Pushbutton, Element Style, and Value Display.

Tools pane — select the Button tool (highlighted) from the Tools section to draw a pushbutton on the canvas.

Configure Pushbutton Animation

Double-click Button1 to open the Edit Animations dialog. Add a Pushbutton animation from the Interaction column. Configure it with Boolean states, reference Cmd, Action set to Set, and On button release checked. This makes the button send a True command to the Cmd property when clicked.

Pushbutton animation configuration — Boolean states, Reference set to Cmd, Action set to "Set", and "On button release" checked. The preview shows the button behavior.

Configure Element Style Animation

Add an Element Style animation from the Visualization column. This animation changes the button's visual appearance based on the Cmd value. When Cmd is True (button active), the element style is set to Active. When Cmd is False (inactive), it uses the Passive style. This provides visual feedback about the current state of the controlled device.

Element Style animation — True/On maps to Active style, False/Off maps to Passive style. This causes the button to visually change color based on device state.

Configure Value Display Animation

Add a Value Display animation with String states and reference CmdOnMsg. This makes the button display the appropriate command text (e.g., "Open" or "Start") from the custom property. Click OK to close the dialog.

Value Display animation — String state with reference CmdOnMsg. The button text dynamically displays the command message from the custom property.

Duplicate and Configure Button2

Right-click Button1 and select Duplicate to create Button2. Move it to the right of Button1, change its text to "Close", then edit its animations: change the Value Display reference to CmdOffMsg, change the Element Style expression to not Cmd (inverting the visual), and change the Pushbutton action to Reset (sending False instead of True).

Two buttons on the drawing canvas — Button1 (Open) on the left and Button2 (Close) on the right. Button2 is selected with blue resize handles visible.
Button2 Value Display — the Expression/Reference field is changed to CmdOffMsg so the Close button displays the off-state command message.

Add Speed Setpoint Display

To support agitator speed control, add an SA_NumericalEntry embedded symbol below the buttons. Configure its Wizard Options to use CustomPropertyLabel for both LabelType and EngUnitsType, and set Outline to False. Then use Substitute References to replace all Me.PV references with Me.Speed.SP. Finally, configure the Label custom property to reference Me.Speed.SP.Description.

SA_NumericalEntry embedded below the buttons — this component provides the speed setpoint entry field for agitator control.
Substitute Reference dialog — replace all occurrences of Me.PV with Me.Speed.SP to bind the numerical entry to the speed setpoint attribute.
Label custom property for SA_NumericalEntry — set to Me.Speed.SP.Description in Expression/Reference mode so the label dynamically shows the speed setpoint description.
Final canvas with fixed dimensions — set Graphic → Size to Fixed with FixedWidth = 320 pixels and FixedHeight = 230 pixels to contain all elements.

Configure FaceplateMode and TreatAsIcon

Set the FaceplateMode property to True on the CommandPanel symbol. This hides the embedded SA_NumericalEntry symbol at runtime when the CommandPanel is used for objects that do not have setpoint attributes (such as valves and pumps). Then open the detailed symbols for Agitator, Pump, and Valve, set their embedded symbols' TreatAsIcon property to True, and add Show Symbol animations pointing to the CommandPanel symbol.

FaceplateMode set to True — this property hides the speed setpoint entry when the CommandPanel is used for equipment without setpoint attributes.
TreatAsIcon property — set to True on the embedded symbol within Agitator_Detailed so it behaves as a clickable icon that triggers the Show Symbol animation.

Configure Show Symbol Animation

For each detailed symbol (Agitator_Detailed, Pump_Detailed, Valve_Detailed), configure a Show Symbol animation that opens the CommandPanel when the embedded symbol is clicked. Use the Galaxy Browser to browse to Graphic Toolbox → Training → Symbols and select CommandPanel. Set the Title to Me.Tagname, Position to Top Left Corner → Mouse, and Type to Modal.

Galaxy Browser — browse to Graphic Toolbox → Training → Symbols and select the CommandPanel symbol as the Show Symbol reference.
Show Symbol animation configuration — reference set to CommandPanel, title set to Me.Tagname, position at Top Left Corner → Mouse, type Modal with close button.
Copy animations — right-click the configured symbol and select Animations → Copy to reuse the Show Symbol configuration on Pump_Detailed and Valve_Detailed.

Test in Preview Session

Switch to the ViewApp preview session and navigate to the Mixer100 process view. Click each equipment symbol to verify that the command panel opens with the correct title, button labels, and visual behavior. Test valve operation (Open/Close), pump operation (Start/Stop), and agitator operation including speed setpoint changes.

Mixer100 process view — the main process diagram shows Pump1_001, Inlet1_001, Pump2_001, Inlet2_001, Agitator_001, Level_001, Temperature_001, and Outlet_001. Click any equipment symbol to open its command panel.
Agitator command panel with speed setpoint — the agitator panel uniquely displays the speed setpoint entry field. The setpoint can be changed by clicking the field and entering a new value.
Editing the speed setpoint — click the speed setpoint field to make it editable, then enter a new value and press Enter to apply.
Speed setpoint updated — after entering 25 rpm, the agitator speed updates both on the command panel and on the Agitator_001 symbol in the main process diagram.

Review Questions

Think about the factors that affected the display of the command panel at runtime:
  • Which configurations determined the text on the buttons? (Answer: the Value Display animation referencing CmdOnMsg and CmdOffMsg custom properties)
  • Why did the visual characteristics of the buttons change in runtime? (Answer: the Element Style animation applies Active/Passive styles based on the Cmd Boolean value)
  • Why did the speed setpoint graphic appear on the agitator panel but not on the pump or inlet? (Answer: the FaceplateMode property hides the SA_NumericalEntry when the object lacks speed setpoint attributes)

📝 Knowledge Check

Q1: What is the purpose of the FaceplateMode property?

FaceplateMode set to True hides embedded symbol elements (like the SA_NumericalEntry speed setpoint) at runtime when the CommandPanel is used for objects that do not have the corresponding attributes (such as valves and pumps).

Q2: Why do you switch to Expression/Reference mode before entering Me.Cmd.OnMsg?

The Default Value field defaults to Static Text mode, which treats input as plain text. Switching to Expression/Reference mode tells the system to evaluate the input as a dynamic reference that resolves to the actual attribute value at runtime.

Q3: What does the Element Style animation do for the buttons?

It changes the button's visual appearance (Active or Passive style) based on the Boolean Cmd value, providing visual feedback about whether the device is in its on or off state.

Q4: How do you reuse the Show Symbol animation configuration across multiple symbols?

Right-click the configured symbol, select Animations → Copy, then open the target symbol, select its embedded element, and choose Animations → Paste to apply the same configuration.
