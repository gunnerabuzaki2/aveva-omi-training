🔬 L53: Lab 26 – Using the ShowContent() Function

Source: AVEVA Operations Management Interface 2023 Training Manual, Module 13 – Scripting in Graphics

Covers creating a ViewApp namespace for tracking layout visibility, configuring layout lifecycle scripts, building a ShowContentButton symbol with Action Scripts, and testing content show/hide functionality. (pages 713–734)

Lab Introduction:

In this lab, you will create a ViewApp namespace and add custom attributes to indicate if a layout is displayed in the ViewApp. You will also create a symbol with buttons configured with animations and scripts to show or close content in the tabbed pane at runtime. Finally, you will update the Main layout with a right slide-in pane and test the toggle behavior.

Content Workflow Highlight: Showing content using the ShowContent() script function
Lab Objectives:
  • Use ShowContent() function to show content in a specific pane
  • Use HideContent() function to close the shown content in a specific pane
  • Use the Layout Predefined scripts
Lab 26 overview: the target layout showing three stacked buttons — Live Alarms, Historical Trend, and Historical Data Grid — in a slide-in pane. These buttons will toggle content visibility in the tabbed pane at runtime.

53.1 Create a ViewApp Namespace

Reference: Module 13, Lab 26 — Create a ViewApp Namespace (page 714)

On the Graphics tab, in the Training \ ViewApp Namespaces toolset, create a new ViewApp namespace named ContentStatus. Double-click to open it in the ViewApp Namespace Editor. Add three Boolean attributes with default settings: HistDataGrid_Displayed, LiveAlarms_Displayed, and HistTrend_Displayed. These attributes will track whether each layout is currently displayed in the ViewApp.

The Graphics tab showing the ViewApp Namespaces section with the newly created ContentStatus namespace and the existing Sources namespace below it.
The ContentStatus ViewApp Namespace Editor with three Boolean attributes configured. All attributes use the default settings — Boolean type with no initial value, suitable for tracking on/off display states.

Save and close the ContentStatus ViewApp namespace, and check it in.

53.2 Configure Layout Scripts

Reference: Module 13, Lab 26 — Layout Scripts Configuration (pages 715–718)

Open the LiveAlarms layout in the Layout Editor and click the Script tab. You'll see three predefined scripts: On Show, While Showing, and On Hide. Configure the On Show script to set the display attribute to True when the layout becomes visible, and the On Hide script to set it to False when the layout closes.

The Layout Editor with the Build and Script tabs visible at the top-left. The red arrow indicates the Script tab where layout lifecycle scripts are configured.

For the LiveAlarms layout, enter the following On Show script:

'Script body begins...
'When the layout is shown, set the ContentStatus.LiveAlarms_Displayed
'attribute to true.
MyViewApp.ContentStatus.LiveAlarms_Displayed = true;
'Script body ends.

And the following On Hide script:

'Script body begins...
'When the layout is hidden, set the
'ContentStatus.LiveAlarms_Displayed attribute to false.
MyViewApp.ContentStatus.LiveAlarms_Displayed = false;
'Script body ends.
The LiveAlarms On Show script configured in the Layout Editor. The script sets MyViewApp.ContentStatus.LiveAlarms_Displayed to true when the layout opens.
The LiveAlarms On Hide script configured in the Layout Editor. The script sets MyViewApp.ContentStatus.LiveAlarms_Displayed to false when the layout closes.

Save and close the LiveAlarms layout, and check it in. Repeat this process for the HistoricalTrend layout, using HistTrend_Displayed instead of LiveAlarms_Displayed:

The HistoricalTrend On Show script — setting MyViewApp.ContentStatus.HistTrend_Displayed to true when the layout opens.
The HistoricalTrend On Hide script — setting MyViewApp.ContentStatus.HistTrend_Displayed to false when the layout closes.

Repeat once more for the HistoricalDataGrid layout, using HistDataGrid_Displayed:

The HistoricalDataGrid On Show script — setting MyViewApp.ContentStatus.HistDataGrid_Displayed to true when the layout opens.
The HistoricalDataGrid On Hide script — setting MyViewApp.ContentStatus.HistDataGrid_Displayed to false when the layout closes.

53.3 Create the ShowContentButton Symbol

Reference: Module 13, Lab 26 — ShowContentButton Symbol (pages 718–721)

Create a new symbol named ShowContentButton in the Training \ Symbols toolset. Open it in the Graphic Editor and add three custom properties:

Property NameData TypeDefault ValueVisibility
ContentDisplayedBooleanFalsePublic
ContentNameString(blank)Public
PaneNameString(blank)Public
The Edit Custom Properties dialog for ShowContentButton. Three properties are defined: ContentDisplayed (Boolean, tracks display state), ContentName (String, identifies content to show), and PaneName (String, identifies target pane).

Add a button element to the drawing canvas with the text "Show Content". The button element is named Button1.

The ShowContentButton symbol canvas with Button1 showing "Show Content" text. This button will be configured with an Action Script to show or hide content.

Double-click Button1 and add an Action Scripts animation. Set the Trigger type to On Left/Key/Touch Up and enter the following script:

'Script body begins...
'Show the target content using ShowContent() function.
Dim contentInfo as aaContent.ContentInfo;
contentInfo.Content = ContentName;
contentInfo.PaneName = PaneName;
'If the content is not yet displayed, then show it in the
'specified pane. Otherwise, hide the content.
if not ContentDisplayed then
  ShowContent( contentInfo );
else
  HideContent( contentInfo );
endif;
'Script body ends.
The Action Scripts configuration for Button1. The script toggles between ShowContent() and HideContent() based on the ContentDisplayed state, providing a toggle-button behavior for content visibility.

Add an Element Style animation to provide visual feedback. Use the Boolean expression ContentDisplayed — when True, use the Active style; when False, use the Passive style.

The Element Style animation configuration. When ContentDisplayed is True, the button uses the Active style (indicating content is shown). When False, it uses the Passive style (indicating content is hidden).

Click OK, save and close the ShowContentButton symbol, and check it in.

📝 Knowledge Check

Q1: What is the purpose of the ContentStatus ViewApp namespace?

It contains Boolean attributes (HistDataGrid_Displayed, LiveAlarms_Displayed, HistTrend_Displayed) that track whether each layout is currently displayed in the ViewApp. These attributes are updated by layout lifecycle scripts.

Q2: How do the On Show and On Hide layout scripts work together?

The On Show script sets the display attribute to true when the layout opens, and the On Hide script sets it to false when the layout closes. This provides a real-time tracking mechanism for which layouts are visible.

Q3: How does the ShowContentButton Action Script toggle content visibility?

It checks the ContentDisplayed property: if False, it calls ShowContent() to display the content; if True, it calls HideContent() to close it. This creates a toggle-button behavior with a single button.

Q4: What is the purpose of the PaneName property on each button?

It specifies which pane (e.g., "Pane8") should receive the content when ShowContent() is called. All three buttons target the same tabbed pane in the bottom-left of the Main layout.

Q5: How do you use Substitute Strings in the Graphic Editor?

Right-click the embedded symbol on the canvas, select Substitute → Substitute Strings, and enter the new text in the New field. This changes the button text without affecting the underlying symbol's scripts and animations.
