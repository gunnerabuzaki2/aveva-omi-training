Section 1 – Historization Overview

AVEVA™ Operations Management Interface 2023

Section 1 – Historization Overview

Introduction

You can historize process data and alarm and event data from the Galaxy with Historian.

In automation objects, you can enable the History feature for attributes that you want to historize.

You can also enable historization for automation objects that have specific history configuration

settings.

To historize process data from automation objects, you must enable the host AppEngine with

storage to historian and specify a Historian server. To historize data from a Platform object, you

must enable the storage to historian on its Engine configuration and specify a Historian server.

Application Server supports the store-and-forward capability for historization.

Historical data can be retrieved and viewed using Historian Client applications, ViewApp

applications with symbols and OMI Apps, Microsoft SQL Server tools, and others.

Historian Client applications provide several client tools to retrieve data and present data in

different formats. The client software includes:



Historian Client Trend



Historian Client Query



Historian Client Workbook (Add-in to Microsoft Excel)



Historian Client Report (Add-in to Microsoft Word)

Historian also provides Historian Client Web, which is a web-based client for data query, trending,

and analysis. With Historian Client Web, you can access historical data stored with Historian

through any supported browser and present the data in different formats.

This section provides an overview of historization and introduces viewing trend data in a

ViewApp.

Trending Data in a ViewApp

Trending real-time data in a ViewApp allows operators to view data within a specified time duration

in a trend format. Current process data is used to update the trend in real time. The real-time trend

can be backfilled with historical data from a historical source.

In a symbol, a Trend Pen element can be added and configured to display a real-time trend.

Historical trends allow operators to view historical data in a ViewApp. Historical data is used to fill

the trend with data from a time period specified as start time, end time, and duration.

For a ViewApp, two OMI Apps are available: the HistoricalTrendApp and the InSightApp.

Security Tracking

Beginning with System Platform 2023, Historian can record OMI user authentications, logons and

logoffs, as well as application starts and stops. To use this feature, Historian must be enabled in

the WinPlatform that hosts the ViewApp.

Section 2 – Real-Time Trending

AVEVA™ Operations Management Interface 2023

Section 2 – Real-Time Trending

Introduction

Real-time trending helps operators monitor process value changes through an updated trend chart

within the visualization client.

A graphic element named Trend Pen can be used to create symbols to display process values as

a trend line that consists of current and historical data updated for a specified interval. The trend

line gives operators a quick visual snapshot of a process value over a period of time.

Additionally, two trend pen symbols are included in the Situational Awareness Library, which are

built using the Trend Pen element.

Trend Pen

The Trend Pen element can be added to a symbol to display trending for the configured reference.

After adding the Trend Pen to the drawing canvas, the Edit Animations dialog box appears, and a

specific animation Trend Pen is listed. Through the animation, you can configure an analog data

source, Historian source, period type, Y axis range, and plot type.

In a running ViewApp, when displaying a symbol with a Trend Pen element configured with a

reference, Historian source, and other settings, a Historian query retrieves data for the entire

Trend Pen over the specified trend period. Before the historical data gets retrieved, the real-time

data is plotted on the trend line. After the historical data is successfully retrieved, it is added to the

trend line to back fill the entire trending period with the real-time data.

This section describes the Trend Pen element for symbols and introduces trend pen symbols

from the Situational Awareness Library.

Situational Awareness Library Trend Pen Symbols

In the Situational Awareness library, the Trends toolset includes two symbols: the

SA_Trend_SinglePen and the SA_Trend_MultiPen symbols.

SA_Trend_SinglePen

SA_Trend_SinglePen can be used to configure a single pen to display real-time and historical data

for a specified reference. There are wizard options available to configure the method to specify the

Historian source, period type, plot type, and more.

For example, the SA_Trend_SinglePen symbol can be configured as either:



Tail Trend: Simple floating pen with basic configurations



Simple Trend: One-pen trend with an optional grid, timescale, and the ability to display a

meter symbol attached with the trend pen

Custom properties are available to configure the pen data source, trend period in minutes, and

others.

The pen elements in the symbol are configured using the predefined element style Trend_Pen.

SA_Trend_MultiPen

SA_Trend_MultiPen can be used to configure multiple pens to display historical data for multiple

references. There are wizard options available to configure the number of pens in the trend, the

methods to specify the Historian source, period type, plot type, the display of a floating tooltip, and

more.

Custom properties are available to configure the data source, line weight, tagname, and visibility

for each pen, selected pen, trend period in minutes, and others.

This symbol supports a maximum of four pens, and their lines are configured using predefined

element styles, which are Trend_PenA, Trend_PenB, Trend_PenC, and Trend_PenD.
