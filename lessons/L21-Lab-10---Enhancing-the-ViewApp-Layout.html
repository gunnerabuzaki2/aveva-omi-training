🔬 L21: Lab 10 – Enhancing the ViewApp Layout

AVEVA™ Operations Management Interface 2023 — Module 4: ViewApp Application Customizations

Pages 309–326 (Lab 10, pp. 4-23 to 4-40)

Lab Objectives
Upon completion of this lab, you will be able to:
  • Configure additional layout properties
  • Configure additional pane properties
  • Enable pan and zoom for panes
  • Configure pane dividers in a layout
  • Use the HamburgerApp

Introduction

In this lab, you will enhance the layout of the ViewApp in several ways. You will add and configure a pane for a hamburger button at the top of the ViewApp, and assign the HamburgerApp to it. The hamburger button (three horizontal lines) opens a slide-in pane on the left side for navigation at runtime. You will also update the slide-in pane so it does not hide the title bar or hamburger button, enable zoom and pan for content panes, configure interactive dividers, and optionally add a background image.

Add Panes for HamburgerApp

Step 1. On the Graphics tab, in the Training \ Layouts toolset, double-click Main to open it in the Layout Editor.

Step 2. In the layout preview area, select Pane2, which has NavBreadcrumbControl assigned.

Step 3. Right-click Pane2 and select Add Pane → Left. A new pane named Pane5 is added to the left of Pane2.

Right-clicking Pane2 to add a new pane to the left for the hamburger button

Step 4. Select Pane5, and on the Properties tab, configure the following size properties:

PropertyValue
Size → Width50
Size → Width SizingFixed
Note: The Size → Height Sizing property for Pane5 is already set to Fixed because this pane was added based on Pane2.
Configuring Pane5 with a fixed 50px width for the hamburger button

Step 5. On the Toolbox tab, enter a search string to find the HamburgerButton and select it.

Step 6. Drag the HamburgerButton to Pane5.

Searching for and dragging the HamburgerButton into the narrow Pane5
Tip: Because Pane5 is very small, you may want to zoom the layout preview display to make it easier to assign content to the pane. You will leave the default configuration for the HamburgerButton, which opens the left slide-in pane when clicked at runtime.

Customize the Left Slide-In Pane

You will add a new pane to the top of the left slide-in pane (Pane3) and customize it to have a transparent background, so that when the slide-in pane is open, the TitleBarControl and HamburgerButton will still be visible.

Step 7. Select Pane3 and add a new pane above it. A new pane named Pane6 is added above Pane3.

Adding Pane6 above Pane3 in the slide-in pane structure

Step 8. Select Pane6, and on the Properties tab, configure the following properties:

PropertyValue
Size → Height131 (same as height of Pane2 + Pane4 + divider)
Size → Height SizingFixed
Size → Hide Row When Emptyunchecked
Pane6 height aligned with the bottom of Pane2 and Pane5 for consistent visual alignment

Set Transparent Background

Step 9. With Pane6 selected, on the Properties tab, click the down-arrow next to the Appearance → Background property to display the color picker. Notice that the color picker shows RGBA color code values, where R is red, G is green, B is blue, and A is alpha (controlling transparency).

The color picker showing RGBA values — the Alpha channel controls transparency

Step 10. Change the Alpha field on the right to a value of 0. This makes the pane background transparent. You can also change the first two digits of the hex code to 00.

Setting Alpha to 0 makes the pane background fully transparent — the hex code updates to #00E4E4E4
Key Concept: RGBA Transparency
The background color uses RGBA hex codes. The first two characters represent the Alpha channel (transparency). Setting Alpha to 00 makes the pane fully transparent. This is essential for overlay panes that should not obscure content behind them, such as the top section of a slide-in pane.

Add Padding to a Pane

Now you will configure padding for a pane, which adds blank-space pixels between the pane border and its content.

Step 11. In the preview area, select Pane1.

Step 12. On the Properties tab, change the Appearance → Padding property to 5,5,5,5.

Step 13. Save and close the Main layout, and check it in.

Adding 5 pixels of padding on all sides of Pane1 to create breathing room around the content

Enable Pan and Zoom

Next, you will customize panes in Asset_Layout. You will enable the pan and zoom controls and configure the divider between panes to be movable at runtime.

Step 14. On the Graphics tab, in the Training \ Layouts toolset, double-click Asset_Layout to open it in the Layout Editor.

Step 15. Ensure Pane1 is selected, and configure its Appearance → Interaction Mode property to PanAndZoom.

Enabling PanAndZoom interaction mode on Pane1 to allow runtime pan and zoom of the process diagram

Make Dividers Interactive

Step 16. Click the pane divider between Pane1 and Pane2, but do not move the divider.

Note: Do not change the position of the divider — just click it to select it and display its properties.

Step 17. On the Properties tab, check the General → Is Interactive property.

Step 18. Save and close Asset_Layout, and check it in.

Enabling the Is Interactive property on the divider allows users to resize panes at runtime

Test the ViewApp in a Preview Session

Step 19. Switch to the ViewApp preview session and ensure you are navigated to Mixer100. Notice that the appearance is different — a hamburger button appears to the left of the NavBreadcrumbControl at the top of the ViewApp.

The new hamburger button appears to the left of the breadcrumb navigation at the top of the ViewApp

Step 20. Click the hamburger button. The left slide-in pane appears. The top pane in the slide-in pane is transparent, so it does not block the title bar or the hamburger button when opened.

Step 21. Click the hamburger button to close the left slide-in pane.

Observe Padding

Step 22. In the display, notice the padding around the content. The padding appears because Pane1 in the Main layout is configured with 5 pixels of padding at the top, bottom, left, and right.

The 5-pixel padding is visible around the content — notice the empty space to the right of the faceplate graphic

Use Pan and Zoom Controls

Step 23. In the pane on the left with the mixer graphic, notice the pan and zoom controls at the bottom of the pane. These appear because the pane is configured with pan and zoom enabled.

Pan and zoom controls at the bottom of the process diagram pane — zoom level, zoom in/out buttons, and pan mode

Step 24. Click the zoom percentage drop-down list and choose 200%.

The zoom dropdown offers magnification levels from 5000% down to 100%

The display zooms in and scroll bars appear on the bottom and right sides of the pane. Notice that the Pan Mode control appears with the zoom controls.

Step 25. Click Pan Mode to enable the pane with pan interaction mode.

Step 26. Use your mouse to click and drag the displayed content within the pane. Notice that the scroll bars move along as you pan.

Step 27. Click Pan Mode again to exit pan interaction mode.

Step 28. Click Zoom To 100% to reset the pane display.

Resetting the zoom to 100% using the Zoom To 100% control

Test Interactive Dividers

Step 29. Move the divider between the pane with the mixer graphic and the pane with the faceplate graphic to the right and left. The divider moves because it is configured with interaction mode enabled.

Step 30. Resize the entire ViewApp window. Notice that the top panes do not change height and the pane with the hamburger button does not change width.

Step 31. When you are finished testing, minimize the ViewApp preview session.

Optional: Add Background Image

The following optional steps demonstrate how to add a tiled background image to the Main layout.

Step 32. On the Graphics tab, in the Training \ Layouts toolset, double-click Main to open it in the Layout Editor.

Step 33. In the preview area, click outside the panes to select the whole layout and display layout properties on the Properties tab.

Step 34. On the Properties tab, for the Appearance → Background Image property, click the ellipsis button to browse for an image file.

Step 35. Navigate to the C:\Training\Other Files folder, select OMITraining_800x800.png, and then click Open. The image appears in the preview area.

Step 36. On the Properties tab, configure the Appearance → Sizing property as Tile.

Note: The Tile effect is not reflected in the Layout Editor preview area. You will use the ViewApp preview session to verify the result.

Step 37. Save and close the Main layout, and check it in.

Step 38. Switch to the ViewApp preview session and navigate to Line1. Notice that the layout background is filled with the tiled image.

Step 39. When you are finished testing, minimize the preview session.

What You Accomplished
In this lab, you added a hamburger button for slide-in navigation, created a transparent overlay pane so the title bar remains visible, added padding for visual breathing room, enabled pan and zoom on process diagrams, and made pane dividers interactive for runtime resizing.

Knowledge Check

Q1: What is the purpose of the transparent Pane6 above the slide-in pane?

Pane6 is transparent so that when the slide-in pane opens, it does not obscure the title bar or hamburger button. The transparency allows content behind it to remain visible.

Q2: How do you enable pan and zoom on a pane?

Select the pane, go to the Properties tab, and set the Appearance → Interaction Mode property to PanAndZoom. This adds zoom controls and pan mode at the bottom of the pane at runtime.

Q3: What does the "Is Interactive" property do for a pane divider?

When checked, the divider can be dragged at runtime to resize the adjacent panes, allowing users to customize the layout on-the-fly.

Q4: Why doesn't the tile background effect show in the Layout Editor?

The Layout Editor does not render the tile effect — it only shows the image once. You must use the ViewApp preview session to verify the tiled background appearance.
