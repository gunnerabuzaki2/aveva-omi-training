Module 7 — ViewApp Namespaces | Section 1 – Introduction

Pages 395–396

Overview

A ViewApp namespace is a repository for ViewApp attributes. You can create references to these attributes in scripts, symbol animations, and OMI Apps to allow attribute values to be shared across different layouts and panes of a running ViewApp. ViewApp namespaces provide a centralized mechanism for storing and distributing data that multiple components within a single ViewApp instance need to access.

Key Concept: You can reuse ViewApp namespaces across multiple ViewApps. However, while configuration data is shared across all ViewApps in the Galaxy, runtime values can only be shared within a single ViewApp. This is because each ViewApp has its own copy of the ViewApp namespace data.

Predefined Namespaces and Attributes

AVEVA OMI provides a set of predefined namespaces and attributes organized by functionality. These built-in namespaces give you access to common system capabilities without needing to create custom namespaces from scratch. For complete details about all predefined namespaces and their attributes, refer to the product documentation.

Referencing ViewApp Namespace Attributes

References to ViewApp namespace attributes must be prefixed with the reserved system name MyViewApp and follow a specific naming convention. The format for a reference is:

MyViewApp.ViewAppNamespace.AttributeName

Here, ViewAppNamespace is the name of a ViewApp namespace available to the Galaxy, and AttributeName is the name of a custom ViewApp namespace attribute defined for that namespace.

Example: A reference to Attribute001 in ViewAppNameSpace_001 would be written as:
MyViewApp.ViewAppNameSpace_001.Attribute001

You can create references to ViewApp namespace attributes in attribute scripts, symbol scripts, and symbol animations. The references can also be used to configure OMI App properties that support reference or expression mode. This flexibility makes ViewApp namespaces a powerful tool for building dynamic, data-driven operator interfaces.

Knowledge Check

Q1: What is a ViewApp namespace?

A ViewApp namespace is a repository for ViewApp attributes. It stores data that can be referenced in scripts, symbol animations, and OMI Apps to share attribute values across different layouts and panes of a running ViewApp.

Q2: Are runtime values shared across multiple ViewApps?

No. While configuration data in a ViewApp namespace is shared across all ViewApps in the Galaxy, runtime values are only shared within a single ViewApp. Each ViewApp has its own copy of the namespace data at runtime.

Q3: What is the correct format for referencing a ViewApp namespace attribute?

The format is MyViewApp.ViewAppNamespace.AttributeName, where MyViewApp is a reserved system name prefix.

Q4: Which namespace is used to control alarm adorner display?

The Alarms namespace is used to determine the display of alarm adorners.
