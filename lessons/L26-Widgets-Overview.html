Section 1 – Introduction

Overview

Widgets are “mini apps” that you can use in your visualization applications. They are reusable

controls designed with web technology and generally display data. They take up more space in the

interface than a single icon and can be interactive and resizable. Widgets are most frequently used

to provide on-screen user interface elements that integrate with other platforms and data sources.

They are supported in Operations Management Interface applications.

Widgets are different from OMI Apps: Widgets are self-contained programs that run automatically

to show information when the ViewApp starts, whereas Apps are typically programs that you must

interact with to run.

Out-of-the-Box Widgets

The following widgets are provided out-of-the-box with Application Server. They are available from

the Graphics tab in Galaxies created with the Default or the Default_EMPTY Galaxy types.



Carousel widget cycles through a series of selected graphics, rotating them at a

configurable, predefined interval. For example, it might be used to display dashboards,

alerts, or alarm information on large monitors on the plant floor.



Datagrid widget shows tabular data within a layout pane of a ViewApp during runtime.

Data shown in the widget can be filtered by the user. The widget also includes a set of

visual properties to customize the appearance of the widget and its displayed tabular data.



MapApp widget shows a map containing symbols within a running application. During

runtime, the map provides controls and touch support to enable users to pan to different

areas of the map and zoom in or out to show more or less map detail. Custom symbols

displayed on the map are user-interactive. For example, clicking a symbol that represents

an asset can be used to automatically navigate the ViewApp to that asset. Symbols shown

on the map typically use latitude and longitude for accurate geo location.

This section introduces using widgets in your ViewApps. It describes the widgets provided out-of-

the-box with Application Server, as well as how to import other widgets into a Galaxy.

Property

Description

Autoplay

If True, the Carousel widget will automatically start on load. If False, the

user must select the next item to start the carousel. The default is True.

BackgroundColor

The background color of the widget.

GraphicNames

A comma-separated list of graphics for the carousel to display at runtime.

Linked graphics that are renamed cannot be displayed with the Carousel

widget. At runtime, they will appear as empty graphics.

Interval

The amount of time (in milliseconds) to delay between automatically

cycling to the next item in the list of graphics to display.

Keyboard

If True, the carousel will respond to keyboard inputs. The default is True.

Loop

If True, the carousel will continuously cycle through the items to display. If

False, the carousel will stop after a single cycle. The default is True.

Pause

If True, the carousel will pause when it detects a mouse-enter or a touch-

down event, and resume cycling of graphics on mouse-leave or touch-up.

If False, the carousel will not pause. The default is True.



QRCode_Scanner widget connects to a camera to scan a QR code and returns the result

as a string. The default value of the string is an empty string.

To use this widget, the device showing the ViewApp must have a camera. It is

recommended for use on a physical machine instead of a virtual machine.

When using the Web Client remotely, use a secure URL (https://).

You can configure a script to read the QR code and display a graphic based on the

scanned value. At runtime, the QRCode_Scanner object will appear with two buttons:

Start Camera and Stop Camera. Users can click Start Camera to scan the QR code. If the

scanned code matches the return value of the script or graphic name, the graphic will

display.



Teamwork widget is a Software-as-a-Service (SaaS) interface that allows workers to

connect to AVEVA Teamwork to perform tasks, achieve learning goals, identify and

manage issues, and capture and document standards. Industrial organizations use

Teamwork to implement skills development, knowledge sharing, and collaboration

management across their enterprise from the cloud. This can be done by using the

Teamwork widget to provide training videos, digital logbook, or for answering calls for

help.



Web_Browser widget can display a web site.

Assign a Widget to a Layout Pane

For Operations Management Interface ViewApps, you can assign widgets to a pane from the

Layout Editor or ViewApp Editor. Select the widget from the Toolbox tab, and then from the

preview area, drag and drop the widget to the pane. Once assigned, you can configure properties

for the widget from the Properties tab.

Property

Description

QRCode

The resulting string of the scanned QR code.

AutoStart

If True, the camera will start on load.

AutoStop

If True, the camera will stop after the QR code is scanned.

StartStop

If True, the camera will start. If False, the camera will stop.

BackgroundColor

The background color of the widget.

Property

Description

URL

The address of a web page to display. The URL must be a valid URL.

If the Web Client is running in HTTPS, then only the HTTPS URL page can

be loaded. If running in HTTP, then both HTTP and HTTPS can be loaded. If

the policy of the web site blocks cross-domain access, then this widget will

not work.

Section 1 – Introduction

Layout Considerations

It will be tempting to lay out your widgets according to the dimensions of the placement grid of a

particular device that you own and develop with. This can be a useful initial approximation as you

lay out your widget, but keep the following in mind:



The number, size, and spacing of cells can vary widely from device to device, so it is very

important that your widget is flexible and can accommodate more or less space than

anticipated.



As the user resizes a widget, the system will respond with a device independent (dp) size

range in which your widget can redraw itself. Planning your widget-resizing strategy

across “size buckets” rather than variable grid dimensions will give you the most reliable

results.

Gestures

Because widgets live on the home screen, they have to co-exist with the navigation that is

established there. This limits the gesture support that is available in a widget compared to a full-

screen App. For example, while Apps may support a view pager that allows the user to navigate

between screens laterally, that gesture is already taken on the home screen for the purpose of

navigating between home panels.

The only gestures available for widgets are touch and vertical swipe.

Import and Export Widgets

You can import a widget into your Galaxy for use in an Operations Management Interface ViewApp

or an InTouch application.

From the Galaxy menu, select Import | HTML5 Widget and then select a .CWP (Custom Widget

Package) file to import. The .CWP file includes HTML5, CSS, and Javascript files.

When the import is complete, the widget will appear in the Galaxy Toolbox and is available for use

in a symbol or a layout. Available widget properties can be configured after the widget is

embedded in a symbol or assigned to a layout pane.

Widgets can also be imported and exported from a Galaxy as objects. In this case, the widget(s)

are imported or exported as aaPKG files.
