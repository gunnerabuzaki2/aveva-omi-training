Introduction to Industrial Graphics

Industrial graphics — also known as symbols — can be used to customize graphic representations of your processes in a ViewApp. They can be embedded in or linked to automation objects, providing an efficient way to visualize object-specific information such as processing status, alarms, history, and more. Symbols support custom properties, relative references, scripts, and other advanced features.

Prebuilt libraries of symbols are provided with System Platform, including the Industrial Graphic Library and the Situational Awareness Library. You can use symbols from these libraries in your applications, or create your own symbols with the Graphic Editor in the System Platform IDE.

Industrial Graphics (Symbols): Customizable graphic assets for visualizing process status, alarms, history, and object-specific data. They can be embedded or linked to automation objects and support custom properties, references, scripts, and more.

The Graphics Tab

The Graphics tab in the System Platform IDE hosts libraries of symbols, controls, profiles, layouts, and other graphic-related content that can be used in the Galaxy. Items available on the Graphics tab depend on which Galaxy template was used to create the Galaxy.

For example, a Galaxy created with the Blank template (based on Default_EMPTY.cab) contains prebuilt symbol libraries, a basic set of OMI Apps, widgets, and controls.

The Graphics tab for a Galaxy created with the Blank template, showing prebuilt libraries including Industrial Graphic Library, Situational Awareness Library, and basic OMI Apps

A Galaxy created with the Default template (based on Default.cab) contains additional prebuilt content including screen profiles, layouts, OMI Apps, widgets, and controls.

The Graphics tab for a Default template Galaxy, showing additional content including Screen Profiles, Layouts, and organized Default/My Content folders

Graphic Toolsets

Symbols, controls, and other items listed in the Graphics tab can be organized into a hierarchy of toolsets. Some toolsets are included by default, depending on the Galaxy type. You can create your own toolsets and sub-toolsets to organize your content as needed.

Graphic toolsets and the organization of items within the Graphics tab are shared by all users of the Galaxy. Top-level graphic toolsets can be hidden or shown in the toolbox as needed, which in turn hides or shows any sub-toolsets. The configuration is associated with the currently logged-in user and saved as part of the Galaxy configuration, allowing users to retrieve their custom views of the Graphics tab from different computers.

Symbol Management

Depending on your development needs, you can create symbols in three locations:

LocationPurposeVisibility
Graphics tabReusable, standard symbolsAvailable system-wide on the Graphics tab
Object templateSymbols for objects derived from that templateOnly within objects derived from the template
Object instanceSymbols for that single instanceOnly within that specific instance
Important: Symbols created within automation objects only exist in those objects and are not listed on the Graphics tab.

Creating Symbols on the Graphics Tab

To create a symbol, right-click the Galaxy name or a toolset name on the Graphics tab and select New > Symbol. The default name is Symbol_00n (where n is the sequential number), but you can rename the symbol. Once created, double-click the symbol to edit it in the Graphic Editor.

Import and Export Symbols

You can export selected symbols or graphic toolsets to an .aaPKG file by selecting them and choosing Galaxy > Export > Object(s). When exporting automation objects that contain symbols, several rules apply:

  • Symbols are exported with their parent automation objects
  • Exporting all automation objects also exports all symbols contained by automation objects and all symbols on the Graphics tab
  • If symbols in an automation object contain embedded symbols from the Graphics tab, these are all exported together
  • Exporting the automation object does not export other automation objects and their symbols
  • Exporting a symbol on the Graphics tab can also export embedded symbols and its host automation object
  • The export process preserves the toolset hierarchy

Symbols can be imported from an .aaPKG file to the Graphics tab using Galaxy > Import > Objects > From package.

Symbol Libraries

Two symbol libraries are provided on the Graphics tab for all Galaxy types:

LibraryDescription
Industrial Graphic LibraryPre-built, realistic-looking symbols of standard industrial objects such as pumps, valves, motors, and heat exchangers
Situational Awareness LibraryPre-built symbols with a simplified look and minimum visual detail, designed to enhance operator situational awareness of current process conditions

Situational Awareness Visualization

Situational awareness visualization provides operators with contextual information to help them determine what actions to take. It can be broken down into three levels:

  1. Perception of new information → Cognition
  2. Comprehension of the information → Judgment
  3. Projection of impact on goals → Decision Making
Situational Awareness is a visual design approach in HMI/SCADA applications that presents operators with contextualized, easily-interpreted information specific to important process issues while filtering out "visual noise." This enables operators to move from perception to comprehension to projection for safer operating conditions.

The benefits of incorporating situational awareness principles include faster problem identification, maximized uptime, accelerated alarm recognition, increased operational consistency, and safer conditions.

Situational Awareness Library Symbols

Symbols in the Situational Awareness Library have a simplified look that efficiently conveys their functional purpose and status. By default, they use a color palette that increases contrast between normal operating conditions (shown in monochromatic colors) and abnormal conditions (shown in vivid colors).

Most Situational Awareness Library symbols are designed as symbol wizards that incorporate multiple visual and functional configurations. Selecting a configuration is done through the symbol's Wizard Options. For example, the SA_Meters symbol can be used as a flow meter, temperature meter, level meter, or other meter types simply by configuring Wizard Options.

Situational Awareness Library symbols include an extensive set of default custom properties that can be set to show or hide parts of the symbol, configure alarm conditions, and show reported values based on the configuration.

Symbols in Automation Objects

When adding symbols into an automation object, you have two options:

  1. Direct Creation: Create symbols directly inside the automation object — these become content of the automation object
  2. Linked Content: Create symbols on the Graphics tab and use the linked content feature to link external symbols to the automation object — linked symbols remain on the Graphics tab but are associated with the automation object as its content
The Object Editor's Attributes tab showing the Link Content button in the Content pane, used to link external symbols from the Graphics tab to an automation object
Tip: You can also use the Link Content icon in the Object Editor to link external content or a layout to an automation object.

Graphic Editor Overview

The Graphic Editor is the tool used to create, configure, and modify symbols. It provides a grid-based canvas on which you can build symbols by adding basic graphic elements (lines, rectangles, curves, etc.) or embedding pre-made symbols from the Toolbox, Assets, or Templates tabs.

The Graphic Editor interface with its five key areas: (1) Toolbars and Menus, (2) Drawing Canvas, (3) Elements List, (4) Element Tools, and (5) Tabbed Area with Properties, Toolbox, Assets, and Templates tabs

The Graphic Editor consists of five key areas:

#AreaDescription
1Toolbars and MenusButtons and menu commands for saving, zooming, line formatting, and more
2Drawing CanvasGrid-based central workspace for placing and arranging elements
3Elements ListHierarchical list of elements on the canvas (left panel)
4Element ToolsCollection of tools for creating symbols — lines, shapes, text, controls (left panel)
5Tabbed AreaProperties, Toolbox, Assets, and Templates tabs for configuring elements and accessing libraries

Canvas Properties

When the canvas is selected, the Properties tab shows properties organized into four categories:

  • Graphic: Symbol description and canvas sizing. Default size is Auto (crops white space). Setting to Fixed uses FixedWidth and FixedHeight for the absolute runtime size.
  • Appearance: Anchor properties (AnchorFixedTo) and smoothing properties (Smoothing).
  • Runtime Behavior: Content types, faceplate mode, scripts, and zoom settings.
  • Custom Properties: User-defined properties for the symbol.

Content Types

A content type can be assigned to a symbol using the ContentType property in the Graphic Editor. The available types match those that can be associated with a layout pane. By default, ContentType is set to None. Selecting All makes the symbol match any type of layout pane.

The ContentType property dropdown in the Graphic Editor showing available content types including Overview, Detailed, Faceplate, Alarm Banner, and others

Embedded Symbols

Embedding a symbol creates a linked copy of the original. If the original symbol is modified, changes propagate to all embedded locations. You can embed symbols from the Industrial Graphic Library, Situational Awareness Library, templates, and instances.

There are three ways to embed a symbol onto the drawing canvas:

  1. Click the Embed Graphic icon from the toolbar, or right-click the canvas and select Embed Industrial Graphic — both open the Galaxy Browser
  2. Drag-and-drop from the Toolbox, Assets, or Templates tabs in the Graphic Editor
  3. Use the search feature in the Toolbox, Assets, or Templates tabs to find symbols by name
The Toolbox tab with "meter" entered in the search field, showing 31 matching content items including AnalogMeter symbols ready to be embedded on the canvas
Tip: Embedding a symbol from a template will initiate a process to create an instance first, and then embed the symbol on the canvas.

Properties of Embedded Symbols

Properties for an embedded symbol are available on the Properties tab when the symbol is selected on the canvas:

  • Graphic: Name of the embedded symbol
  • Appearance: X and Y location, size, orientation, anchor points, and scaling
  • Runtime Behavior: Visibility, tab order, symbol references, container status, and the OwningObject property
  • Custom Properties: User-defined public properties from the embedded symbol

The OwningObject Property

The OwningObject property is a runtime property used to dynamically change the references within an embedded symbol from one instance to another. It replaces all Me. references in expressions and scripts. The object name can be set using the tagname or hierarchical name of an automation object.

The OwningObject property being configured to reference Agitator_002, dynamically rebinding the embedded symbol to a different automation object instance

Symbol Wizard Options

If a symbol with Wizard Options is selected on the drawing canvas, its Wizard Options appear on the Properties tab. Many Situational Awareness Library symbols have Wizard Options pre-configured for easy configuration of functional modes.

The Properties tab showing Wizard Options for the SA_Meters1 symbol, with configurable options for Type, Orientation, Label, and display settings

Custom Properties Configuration

Custom properties can be configured through the Edit Custom Properties dialog box (right-click an embedded symbol and select Custom Properties). Each property has a defined data type, default value, visibility setting, and description. The default value can be overridden with a value, reference, or expression.

The Edit Custom Properties dialog showing properties like EngUnitsVisible, LabelVisible, and PV with their data types, default values, and visibility settings

Substitute Strings and References

You can search for and replace strings of any element on the canvas that has the Text property using Special > Substitute Strings or the right-click context menu. This supports basic list-based replacement as well as advanced Find & Replace with Match Case, Wildcards, and other options.

The Substitute Strings dialog showing Old/New columns for replacing text strings, with Find & Replace options for advanced search

Similarly, you can search for and replace reference paths on canvas elements using Special > Substitute References. The Galaxy Browser can be used to select new references, though auto-complete is not available in this dialog.

The Substitute Reference dialog showing Old/New columns for replacing reference paths like Me.PV, with Find & Replace capabilities
Key Takeaway: The Graphic Editor is the central tool for creating and customizing industrial graphics in System Platform. It supports embedding linked copies of prebuilt symbols, configuring content types for ViewApp integration, using Wizard Options for rapid symbol configuration, and substituting strings and references for customization.

Quiz

Test Your Knowledge

1. What are the two prebuilt symbol libraries provided with System Platform?

The Industrial Graphic Library (realistic, detailed symbols of standard industrial equipment) and the Situational Awareness Library (simplified symbols designed for quick process condition interpretation with minimal visual detail).

2. What are the three locations where symbols can be created, and how do they differ?

Symbols can be created on the Graphics tab (reusable, system-wide), in an object template (available to objects derived from that template), or in an object instance (available only in that specific instance). Symbols created within automation objects do not appear on the Graphics tab.

3. What happens when you embed a symbol from the Situational Awareness Library?

Embedding creates a linked copy of the original symbol. If the original symbol is modified, changes propagate to all embedded locations. Custom edits to the embedded copy do not affect the source symbol. The embedded symbol can be customized through properties, Wizard Options, and custom properties.

4. What is the purpose of the OwningObject property?

The OwningObject property is a runtime property that dynamically rebinding an embedded symbol to a different automation object instance. It replaces all "Me." references in expressions and scripts, allowing the same embedded symbol to display data from different instances.

5. What is the difference between the Substitute Strings and Substitute References features?

Substitute Strings searches for and replaces text content of elements with the Text property (like chart headings and labels). Substitute References searches for and replaces reference paths on graphic elements (like Me.PV or Me.Level.PV). Both support basic list-based replacement and advanced Find & Replace functions.
