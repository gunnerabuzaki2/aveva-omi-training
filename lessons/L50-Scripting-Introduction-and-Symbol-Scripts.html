📖 L50: Scripting Introduction and Symbol Scripts

Source: AVEVA Operations Management Interface 2023 Training Manual, Module 13 – Scripting in Graphics, Sections 1–2

Covers the scripting environment for OMI graphics, symbol scripts (predefined and named), script triggers, and the Action Scripts animation. (pages 687–692)

50.1 Introduction to Scripting in Graphics

Reference: Module 13, Section 1 — Introduction (page 687)

Operations Management Interface allows you to add scripts to symbols or their elements to animate the symbols or modify their elements while a ViewApp is running. You can also add scripts to layouts to customize the runtime behavior of the layouts and layout content. The scripting language used is QuickScript .NET.

What Scripts Can Do in OMI Graphics:
  • Perform assignments, comparisons, simple math functions, and similar actions
  • Include logical operations using conditional branching and code looping
  • Reference attributes of automation objects, properties of elements and embedded symbols
  • Reference custom properties and attributes of the ViewApp
  • Define and use local variables within scripts
  • Call runtime methods such as ShowLoginDialog()

For example, you can use the ShowLoginDialog() method in a script so that when the script is executed at runtime, it will prompt with the login dialog box if security in the Galaxy is enabled. This demonstrates how scripts can interact with runtime features beyond simple data manipulation.

50.2 Execution Order of Symbol and Layout Scripts

Reference: Module 13, Section 1 — Execution Order (page 687)

When a ViewApp is running, symbol scripts and layout scripts execute in a specific order designed to ensure that all references exist when a layout script calls them. In general, symbol scripts run before layout scripts. However, there are important exceptions to understand.

Execution Priority Rules:
  1. The On Hide layout script has precedence over symbol scripts — it runs first when the layout closes
  2. Symbol scripts have precedence over the On Show layout script
  3. Named layout scripts can run in any order and may be interspersed with symbol scripts
  4. Any named script triggered by the Data Change trigger type runs the first time the reference is subscribed to
Important Difference from Application Server: The Data Change trigger behavior in OMI graphics scripts is different than in Application Server scripts. In OMI, a Data Change triggered script runs the first time when the reference is subscribed to, which can take considerable time in intermittent networks. This is a key distinction to keep in mind when writing scripts.

50.3 Symbol Scripts — Overview

Reference: Module 13, Section 2 — Symbol Scripts Introduction (page 689)

You can add scripts to symbols to animate the symbols or modify their elements while a view application is running. Scripts are added to a symbol using the Edit Scripts dialog box, which you access from the Graphic Editor by right-clicking the drawing canvas and selecting Scripts.

The Edit Scripts dialog box for a symbol. The left pane shows the Symbol Scripts tree with Predefined Scripts and a + button to add new scripts. The right pane shows the Predefined Scripts configuration with trigger settings and an empty script editor area.

With symbol scripts, you can configure the predefined scripts for a symbol, add named scripts, edit or rename existing scripts, remove named scripts, substitute references in scripts, and use element methods. This flexibility makes symbol scripts a powerful tool for creating dynamic, interactive graphics.

50.4 Predefined Symbol Scripts

Reference: Module 13, Section 2 — Predefined Symbol Scripts (page 690)

Predefined symbol scripts run based on the display state of the symbol in the running view application. There are three predefined script triggers, each tied to a specific moment in the symbol's visibility lifecycle.

TriggerWhen It RunsDescription
On ShowOnce when symbol opens or becomes visibleFires once after the symbol is shown in the running ViewApp. Can configure a Data Timeout in milliseconds.
While ShowingPeriodically while symbol is visibleRuns at a specified interval while the symbol appears in the running ViewApp.
On HideOnce when symbol closes or becomes hiddenFires once after the symbol is closed or hidden. No configurable trigger parameters.

50.5 Named Symbol Scripts

Reference: Module 13, Section 2 — Named Symbol Scripts (page 690)

Named scripts are custom scripts that you create by clicking the Add Script button. Unlike predefined scripts, named scripts run based on an expression or reference that you define. The expression in the Expression field acts as a data source for the script trigger.

TriggerWhen It Runs
WhileTrueRuns periodically while the expression is true
WhileFalseRuns periodically while the expression is false
OnTrueRuns once on a false-to-true transition
OnFalseRuns once on a true-to-false transition
DataChangeRuns when the data value or quality state changes

50.6 Action Scripts Animation

Reference: Module 13, Section 2 — Action Scripts Animation (page 691)

Action Scripts are a special type of animation that you can add to a graphic element using the Edit Animations dialog box. From the Graphic Editor, double-click an element on the drawing canvas, click the Add Animation button, and then select Action Scripts. This allows you to attach script execution directly to user interactions with specific elements.

At runtime, action scripts execute one time or periodically when an operator performs a mouse action. Available triggers include On Left Click, While Left, On Left Double-Click, and many others. One or more action scripts can be added to a single graphic element. Additionally, action scripts can be added for an embedded symbol, but the embedded symbol must have its TreatAsIcon property set to True.

Script Development Tools: When writing script code, you can use the Script Function Browser, Graphic Browser, Automation Object Browser, and Attribute Browser to select external data. The script editor supports autocomplete to help you find the right attributes and methods. The Clear Value icon removes all code, and the Validate Value icon checks for warnings or errors.

50.7 Predefined Script Configuration

Reference: Module 13, Section 2 — Predefined Scripts Configuration (page 692)

Each predefined script type has additional configuration options. For example, when the trigger type is On Show, you can configure the Data Timeout in milliseconds. This timeout determines how long the script will wait for a referenced attribute to become available before stopping execution. This is particularly useful in environments where data sources may not be immediately available when a symbol first appears.

The predefined scripts window shows the trigger type, data timeout setting, the number of scripts used, and provides a script editor area where you can write the actual code. The editor toolbar includes icons for saving, undoing, redoing, and executing scripts.

Lesson Summary

Key Concepts Covered:
  • QuickScript .NET is the scripting language used for OMI graphics scripts
  • Scripts can be added to symbols (via Edit Scripts) or layouts (via the Layout Editor Script tab)
  • Predefined symbol scripts run based on display state: On Show, While Showing, On Hide
  • Named symbol scripts run based on expressions: WhileTrue, WhileFalse, OnTrue, OnFalse, DataChange
  • Action Scripts execute when operators interact with elements (clicks, double-clicks, etc.)
  • Execution order: Symbol scripts run before layout scripts, except On Hide which runs first
  • The Data Change trigger in OMI runs the first time a reference is subscribed to — different from Application Server behavior

📝 Knowledge Check

Q1: What scripting language is used for OMI graphics scripts?

QuickScript .NET. It supports assignments, comparisons, math functions, conditional branching, and code looping.

Q2: What are the three predefined symbol script triggers?

On Show (runs once when symbol becomes visible), While Showing (runs periodically while visible), and On Hide (runs once when symbol is hidden).

Q3: What is the general execution order of symbol scripts versus layout scripts?

Symbol scripts run before layout scripts. The exception is the On Hide layout script, which has precedence over symbol scripts and runs first when the layout closes.

Q4: What type of trigger does a named symbol script use?

Named scripts use expression-based triggers: WhileTrue, WhileFalse, OnTrue, OnFalse, or DataChange. The expression in the Expression field acts as the data source for the trigger.

Q5: How is the Data Change trigger behavior in OMI different from Application Server?

In OMI, a Data Change triggered script runs the first time the reference is subscribed to. This behavior differs from Application Server and can take considerable time in intermittent networks.
