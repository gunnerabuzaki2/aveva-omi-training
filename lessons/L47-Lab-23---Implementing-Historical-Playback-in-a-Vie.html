🔬 L47: Lab 23 – Implementing Historical Playback in a ViewApp

AVEVA™ Operations Management Interface 2023 — Module 11: Historical Playback

Pages 643–666 (Lab 23)

Lab Objectives: Upon completion of this lab, you will be able to:
  • Configure the historical data replay service in the Galaxy
  • Use prebuilt symbols for historical playback functions in a running ViewApp
  • View historical data in playback mode in a running ViewApp

Introduction

In this lab, you will implement the historical playback feature in your ViewApp. You will start by configuring the historical data playback service in your Galaxy. Then, you will import some prebuilt symbols, including a playback control panel, a playback status indicator, and a playback dashboard symbol. You will update the Main layout with a slide-in pane on the bottom and assign the playback control panel symbol to it. You will also add a small pane at the top and assign the playback status indicator symbol to it.

Then, in the ViewApp, you will add a navigation item to access the playback dashboard symbol and test historical playback in a preview session.

Security Note: The playback control panel symbol is configured such that only users with access levels greater than 9000 can start historical playback.
Key Concept — Historical Playback Access Levels: Enabling historical playback requires a user access level greater than 9000. Users in roles with lower access levels (e.g., Plant Operators at level 2000) will be unable to activate playback mode. The control panel displays a floating tooltip showing the security requirement when hovered, and the switch simply refuses to activate for insufficient access levels.
The SCADA Playback control panel symbol with the enable switch, From/To time range fields, play/pause/next/previous buttons, and current time display

Part 1: Create and Deploy the Playback Service

First, you will create an instance of the historical playback service and rename the service instance with the name of the Historian node. Then you will deploy the instance.

Key Concept — SCADA Playback Service: Historical playback requires the ASB.HistoricalDataReplayService.1.0 service to be created, assigned to nodes, and deployed via Configure > System > Services. The service instance is typically renamed to match the Historian server hostname (e.g., S00ENG). As long as one node in the Galaxy runs this service, any ViewApp in the Galaxy can use playback.
  1. On the Galaxy menu, select Configure | System | Services.
Navigate to Configure > System > Services in the System Platform IDE backstage to open the Configure Services dialog

The Configure Services dialog box appears.

  1. In the left pane, expand TrainingGalaxy.
  2. Right-click ASB.HistoricalDataReplayService.1.0 and select Create.
Right-click ASB.HistoricalDataReplayService.1.0 and select Create to generate a new service instance

An instance of the service named HistoricalDataReplayService_001 is created. You may need to expand ASB.HistoricalDataReplayService.1.0 to see the new instance in the list.

  1. Rename the service instance to the name of the Historian node in your Galaxy.
Note: Your instructor will provide you with the name of the Historian node. In the following example, S00ENG is used as the name of the service instance.
The newly created HistoricalDataReplayService_001 instance appearing under ASB.HistoricalDataReplayService.1.0
After renaming to S00ENG, the Assignments pane shows available nodes (S00ENG and S00PROD) for service deployment
  1. In the left pane, ensure that the service instance is selected.
  2. In the bottom-right pane under Assignments, check the node(s) for the assignment.

You can assign the service to one or more nodes in the Galaxy. As long as there is a node in the Galaxy running the deployed service, the playback feature can be used for any running ViewApp in the Galaxy. It is better to deploy the playback service on the nodes on which the ViewApp will be running and using the playback feature.

Note: Your instructor will provide information about which node or nodes to select under Assignments. In the example below, S00ENG and S00PROD are selected.
  1. Click Update. A message on the bottom-left status bar indicates that the assignment operation was successful.
The status bar confirms: "S00ENG Assign operation successful" and "S00PROD Assign operation successful"
  1. In the left pane, right-click the service instance and select Deploy. When deployment is complete, the icon changes to indicate it is deployed and a Deploy Success message appears in the status bar.
  2. Close the Configure Services dialog box.
Right-click the renamed service instance and select Deploy (Ctrl+D) to deploy it to the assigned nodes
After deployment, the status bar shows "Deploy Success" for both S00ENG and S00PROD nodes

Part 2: Import Playback Content

In the following steps, you will import three prebuilt symbols: Playback_ControlPanel, Playback_Status, and Playback_Dashboard.

  1. Ensure you are still in the backstage and select Import | Objects | From package.
  2. Navigate to C:\Training, select Lab 23 - Content for Playback_Prebuilt.aaPKG, and then click Open.
  3. Leave the default settings and click Import.
In the backstage, select Import > Objects > From package to import the prebuilt playback symbols
  1. When the status of the import process is complete, click Close, and then return to the main page of the System Platform IDE. On the Graphics tab, notice that three new symbols are imported into the Training \ Symbols toolset: Playback_ControlPanel, Playback_Dashboard, and Playback_Status.
Three new playback symbols visible on the Graphics tab: Playback_ControlPanel, Playback_Dashboard, and Playback_Status

Part 3: Update the Main Layout

In the following steps, you will update the Main layout to add slide-in panes for the playback controls.

  1. On the Graphics tab, in the Training \ Layouts toolset, double-click Main to open it in the Layout Editor.
  2. At the bottom of the layout, click the Add Slide-In Pane button. A slide-in pane named Pane9 is added at the bottom.
Pane9 added as a slide-in pane at the bottom of the Main layout using the Add Slide-In Pane button
  1. On the Properties tab, click the down-arrow to the right of the Appearance \ Background property to display the color picker.
  2. In the list of RGBA color code value fields, change the last field (alpha) to a value of 0 for full transparency.
Setting the Background alpha channel to 0 makes Pane9 fully transparent, allowing the playback panel to float over the content
  1. For the Appearance \ Content Alignment property, change the value to Bottom.
  2. On the Toolbox tab, enter a search string to find the Playback_ControlPanel symbol, and drag it to Pane9.
Content Alignment property set to Bottom ensures the playback control panel anchors to the bottom of Pane9
The Playback_ControlPanel symbol is now assigned to Pane9, appearing as a slide-in panel at the bottom of the layout
  1. Select Pane2, and then right-click and select Add Pane | Right. A new pane named Pane10 is added to the right of Pane2.
  2. Select Pane10.
  3. On the Properties tab, change the Size \ Width property to 170. The width of Pane10 is reduced in the preview.
Right-click Pane2 and select Add Pane > Right to create a narrow Pane10 for the status indicator
  1. On the Toolbox tab, enter a search string to find the Playback_Status symbol, and drag it to Pane10.
  2. Save and close the Main layout, and check it in.

Part 4: Add a Navigation Item

In the following steps, you will add a navigation item to the ViewApp and configure it to access the Playback_Dashboard symbol.

  1. Switch to the ViewApp Editor.
  2. Ensure Home is selected in the Navigation list.
The ViewApp Editor with Home selected in the navigation tree; the main layout now shows the playback control panel at the bottom
  1. Click Add to add a new navigation item, and name it PlaybackDashboard.
  2. On the Properties tab, update its Navigation \ Title property to Playback Dashboard.
The new PlaybackDashboard navigation item added under Home, with the Navigation Title property set to "Playback Dashboard"
  1. On the Toolbox tab, enter a search string to find the Playback_Dashboard symbol, and drag it to Pane1.
The Playback_Dashboard symbol assigned to Pane1 of the PlaybackDashboard navigation item, displaying live totalizer data for each mixer

Now you will update the preview session to test the new navigation item and historical playback feature.

  1. Click Update to update the preview session. The preview opens navigated to Playback Dashboard.

Notice that <domain>\jamess is still logged in. Recall that this user is in the Application Administrators role, which is configured with an access level of 9999. Also notice the Display Slide-In Pane button at the bottom of the display.

  1. At the bottom of the display, click the Display Slide-In Pane button to show the Playback_ControlPanel symbol. Notice the Historical Playback Enable switch shows that historical playback mode is not enabled.
The Playback Dashboard showing live data; the Display Slide-In Pane button is visible at the bottom of the screen
The Playback ControlPanel with the Historical Playback Enable switch showing playback mode is currently disabled
  1. Hover your mouse over the Historical Playback Enable switch. A floating tooltip displays the security access level requirement (access level higher than 9000).
  2. Click the Historical Playback Enable switch to enable historical playback mode. Additional controls appear on the Playback_ControlPanel symbol. The Playback_Status symbol in the top-right blinks to indicate Playback Mode Active.
After enabling playback, the control panel expands with time range fields and playback buttons; the status indicator blinks "Playback Mode Active"
  1. In the Playback_ControlPanel symbol, click the From time to input an initial time to start the playback.
Clicking the From time field opens the Time User Input dialog for setting the playback start time

The Time User Input dialog box appears.

Note: This dialog box shows time in 24-hour format.
  1. In the Time area, increase the Hour field by seven hours.
  2. Click OK.
  3. In the Playback_ControlPanel symbol, click the Play button to start historical playback.
After clicking Play, the Playback Dashboard begins displaying historical data; the progress bar circle starts moving to the right

Notice the Playback_Dashboard symbol starts displaying historical data. At the top of the Playback_ControlPanel symbol, the circle in the history time progress bar is moving to the right.

  1. Click the circle in the history time progress bar and drag it to the right. Floating tooltips show the date and time of the historical data based on the position of the progress bar.
Dragging the progress bar circle shows floating tooltips with timestamps, allowing you to scrub through historical data
  1. On the right side of the Playback_ControlPanel symbol, slide the Speed slider to the right and to the left, to see the playback speed up and slow down.

Part 6: Test Live Value Playback Icon

In the next steps, you will disable historical playback mode, navigate to a different view, and then enable playback mode again to observe the Live Value Playback icon.

Key Concept — Live Value Playback Icon: The green triangle icon appears over graphics during historical playback to indicate that some displayed values are from live (non-historized) sources. This informs operators that not all data references can be resolved from historical data — for example, engineering unit labels or calculated values may still show current values rather than historical ones.
  1. Click the Historical Playback Enable switch to disable playback.
  2. Click off the bottom slide-in pane to close it, and then navigate the ViewApp to Mixer100.
Disabling playback mode via the Historical Playback Enable switch before navigating to a different asset
  1. Click the Display Slide-In Pane button to show the Playback_ControlPanel symbol again.
  2. Click the Historical Playback Enable switch to enable playback. A Live Value Playback icon (green triangle) appears over graphics that contain non-historized values during historical playback. This icon informs the viewer that not all references used in the graphics can be resolved from historical sources.
The green Live Value Playback icon appears above the Mixer100 graphic, indicating that some displayed values (like engineering units) are from live sources

  1. When you are finished testing, close the preview session and close the ViewApp Editor.

Knowledge Check

Q1: What service must you create and deploy before using historical playback?

You must create and deploy an instance of the ASB.HistoricalDataReplayService.1.0 from the Configure Services dialog box, and name it to match the Historian server hostname.

Q2: What three prebuilt symbols are imported for the playback feature?

Playback_ControlPanel (the control bar with play/pause/speed controls), Playback_Status (the blinking status indicator), and Playback_Dashboard (the data display symbol).

Q3: Why must Pane9's Background alpha be set to 0?

Setting the alpha channel to 0 makes the slide-in pane fully transparent, allowing the playback control panel to appear as a floating overlay on top of the main content.

Q4: What does the Live Value Playback icon (green triangle) indicate?

It indicates that some displayed values on the graphic are from live (non-historized) sources, because not all data references can be resolved from historized data during playback.

Q5: Why can't user "johnj" enable historical playback?

Johnj belongs to the Plant Operators 1 role with an access level of 2000, but enabling playback requires an access level greater than 9000.
