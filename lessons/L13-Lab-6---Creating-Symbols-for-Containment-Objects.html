🔬 L13: Lab 6 – Creating Symbols for Containment Objects

AVEVA™ Operations Management Interface 2023 — Module 3, Lab 6

Pages 213–234 (Section 3-75 to 3-96)

Lab Introduction

In this lab, you will create a comprehensive symbol within the $Mixer template — a container template that holds multiple child objects such as pumps, valves, agitators, level meters, and temperature sensors. The symbol you build will embed symbols from all contained objects using relative references, add text labels, draw connectors between equipment, and apply element styles for visual consistency.

Lab 6 overview — the target result is a complete mixer process diagram showing two pumps feeding into a mixer through inlet valves, with an agitator, level meter, and temperature sensor inside the mixer tank, and an outlet valve on the discharge side. (Section 3-75)
Lab Objectives
Upon completion of this lab, you will be able to:
  • Embed symbols from contained objects using relative references
  • Configure and embed symbols within a containment relationship
  • Add Text elements and configure Value Display animations
  • Apply element styles to elements for visual consistency
  • Use Connection Points and Connectors to draw process flow lines
  • Change element display orders using the Elements list
  • Configure the drawing canvas with a fixed size

Part 1: Creating a Symbol in $Mixer

Reference: Pages 214–215 (Section 3-76 to 3-77)

Steps 1–4: Create and Name the Symbol
On the Templates tab, navigate to Training \ Project and double-click $Mixer to open it in the Object Editor. On the Attributes tab, in the Content pane, click Add to create a new symbol. Rename it to MixingProcess_Detailed and add the description "Includes symbols from all contained objects."
Content pane before adding a symbol — the pane is empty with instructions to "Press one of the '+' buttons above to create or link content." The Add button (left "+" icon) creates a new symbol. (Section 3-76)

Steps 5–6: Open and Configure the Symbol
With MixingProcess_Detailed selected, click Edit Content to open it in the Graphic Editor. On the Properties tab, set the Runtime Behavior \ ContentType property to Detailed.
Content pane with the Edit Content button (pen icon) highlighted — clicking this opens the selected symbol in the Graphic Editor. Alternatively, you can double-click the symbol. (Section 3-77)
Properties tab in the Graphic Editor — ContentType is set to "Detailed" under Runtime Behavior. This ensures the symbol is displayed when users navigate to the Mixer equipment level. (Section 3-77)

Part 2: Embedding Symbols via Relative References

Reference: Pages 216–217 (Section 3-78 to 3-79)

Key Concept — Embedding via Relative References
The Galaxy Browser supports embedding symbols from relative references. When you click "Embed Industrial Graphic" in the toolbar, the Galaxy Browser opens with a Relative References tab. This tab lists all objects that are children of the current object (e.g., Me.Agitator, Me.Inlet1, Me.Level, etc.). You select a relative reference, then choose which symbol from that object to embed. This creates a live link — the embedded symbol will display data from the corresponding contained object at runtime.
Steps 7–10: Embed the Agitator Symbol
In the Graphic Editor toolbar, click Embed Industrial Graphic. In the Galaxy Browser, click the Relative References tab. Select Me.Agitator in the left pane, then select Agitator_Detailed in the right pane and click OK.
Graphic Editor toolbar — the "Embed Industrial Graphic" button (highlighted with red arrow) opens the Galaxy Browser to embed symbols from relative references or the Graphics tab. (Section 3-78)
Galaxy Browser with the Relative References tab active — this tab lists all objects relative to the current $Mixer template: Me, Me.Agitator, Me.Inlet1, Me.Inlet2, Me.Level, Me.Outlet, Me.Pump1, Me.Pump2, and Me.Temperature. (Section 3-78)
Galaxy Browser with Me.Agitator selected — the right pane shows the Agitator_Detailed symbol preview. Click OK to embed this symbol into the MixingProcess_Detailed canvas. (Section 3-78)

Click the drawing canvas to place the agitator graphic. Then repeat Steps 7–11 to embed all remaining symbols. Arrange them on the canvas as shown in the layout diagram.

Part 4: Adding Connection Points and Connectors

Reference: Pages 219–225 (Section 3-81 to 3-87)

Connection Points and Connectors are essential for creating process flow diagrams. Connection Points define where connectors attach to elements, while Connectors draw the lines between them.

Key Concept — Connection Points and Connectors
Connection Points are snap points placed on element borders where connectors can attach. When you hover over an element border with a Connection Point tool active, the border turns green to indicate a valid placement location. Connectors are lines drawn between connection points — click and drag from one connection point to another to create the connector. The connector automatically snaps to the connection points.
Steps 21–25: Place Connection Points on the Tank
In the Tools pane, double-click Connection Point. Place three connection points on the tank border: two on the left side (aligned with Inlet1 and Inlet2) and one on the right side (aligned with Outlet). The tank border turns green when the cursor hovers over it.

Quiz

Question 1

Why do you use the Relative References tab in the Galaxy Browser instead of browsing the Graphics tab when embedding contained symbols?

Click to reveal answer

The Relative References tab provides direct access to the child objects of the current container (e.g., Me.Agitator, Me.Inlet1, Me.Level). This ensures the embedded symbol is linked to the correct contained object through a relative reference that will resolve automatically at runtime. Browsing the Graphics tab would link to a static symbol rather than a dynamically-resolved reference to the contained object's data.

Question 2

What is the purpose of the "Send To Back" command when working with the SA_Tank_Vessel element?

Click to reveal answer

The SA_Tank_Vessel is placed on the canvas after the embedded equipment symbols, so it initially appears in front of them (covering the agitator, level, and temperature symbols). "Send To Back" moves it to the bottom of the z-order, placing it behind all other elements so the contained equipment symbols are visible on top of the tank.

Question 3

Why is it important to align Connection Points with the built-in connection points on adjacent elements?

Click to reveal answer

Connection Points must align with the built-in connection points on adjacent elements so that Connectors can snap to both points cleanly. If they are misaligned, the connector line may appear at an angle or not connect properly. The green border highlight when hovering confirms correct alignment.

Question 4

What does the Value Display animation with the "Name" state and "Tag Name" option display at runtime?

Click to reveal answer

It displays the object's tag name (the short name used in the Galaxy hierarchy) as text within the element. For example, when viewing Agitator_001, the ObjectName element would display "Agitator_001". When viewing Mixer100, it would display "Mixer100". This makes the symbol dynamic — it automatically shows the correct name for whatever object is currently being displayed.

Key Takeaways

  1. Embedding via Relative References creates live links to contained objects that resolve automatically at runtime.
  2. The Elements list z-order controls which elements appear in front — use "Send To Back" to layer background elements behind foreground content.
  3. Connection Points define snap locations on element borders; Connectors draw process flow lines between them.
  4. Value Display animations make text elements dynamic — they show object names, areas, or data values at runtime.
  5. ElementStyles (Title, Heading, etc.) ensure visual consistency across all labels in your Galaxy.
  6. Substitute Strings allows bulk text replacement in element properties.
  7. Fixed canvas size ensures consistent symbol dimensions across all instances.
  8. A single template symbol with relative references works for all derived objects without additional configuration.
