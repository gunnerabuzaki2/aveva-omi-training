📖 L41: Lab 20 – Creating a Historical Alarm Display

Module 9 — Alarms and Events Visualization, Section 3

Pages 573–600

Objectives

Upon completion of this lab, you will be able to:

  • Configure the AlarmApp to display historical alarms and events
  • Use the AlarmApp to query logged alarms and events at runtime
  • Customize the AlarmApp at runtime using Runtime Behavior properties

Configure Layouts for Historical Alarms and Events

Create a new layout named HistoricalAlarmsEvents in the Training > Layouts toolset. Open it in the Layout Editor and add the AlarmControl to Pane1 from the Toolbox tab.

The HistoricalAlarmsEvents layout in the Layout Editor with the AlarmControl widget placed in Pane1 via drag-and-drop from the Toolbox

Configure the Historical Alarm Database

Click the ellipsis button next to the Configuration property. On the Alarm Mode tab, set Client Mode to Historical Alarms. Enter your Historian server name for Database Connectivity > Server Name, and select History Blocks from the Database Name dropdown. A configuration warning will appear — click Yes to continue.

Properties tab with Run-Time Behavior section visible and the Configuration ellipsis button highlighted with a red arrow
Configuration Dialog — Alarm Mode tab with Client Mode "Historical Alarms", server name entered, and Database Name set to WWAlmDb before switching to History Blocks
Configuration Dialog showing History Blocks selected as the database, and the success Info dialog: "Connection to History Blocks Data Storage succeeded"

Configure Time Range and Column Details

Set the Time Range duration to 15 minutes. On the Column Details tab, uncheck columns that are unnecessary for historical display: User1, State, Node, Group, Name, AlarmComment, Type, TimeLCT, Limit, CurrentValue, AlarmDuration, Operator, and UnAckDuration. Then check columns useful for historical alarms: Severity, Comment, Provider_Nodename, Provider_ApplicationName, User Account, User Name, Source_Area, Source_ConditionVariable, Source_Object, Source_ProcessVariable, Alarm_Type, Alarm_Condition, Alarm_LimitString, Alarm_State, Verifier Account, Verifier Name, and EventTime.

Other Settings section showing Time Range dropdown with 15 minutes selected, Maximum Records set to 100, and Update to Current Time checked
Column Details tab with EventTime selected and the up-arrow button to move it above Comment and below Severity in the display order

Set the First Sort Criteria to EventTime and Descending. Change the Comment row's Width to 300 pixels. On the Run-Time Behavior tab, uncheck Save User Queries and Filters.

Column Details with Comment width being changed to 300; Run-Time Behavior tab with "Save User Queries and Filters" unchecked

Configure Command Bar Properties

On the Properties tab, configure the Command Bar: uncheck Show Command Bar, Show Ack Selected, Show Ack All, and Show Ack Visible; check Show Connect and Show Disconnect. Save and close the HistoricalAlarmsEvents layout, and check it in.

Command Bar section with Show Command Bar unchecked, Ack buttons unchecked, and Connect/Disconnect checked — these buttons are relevant for historical data operations

Create a Symbol for Additional Settings

Create a new symbol named Hist_AlarmAppConfigurations in the Training > Symbols toolset. Open it in the Graphic Editor and create the following custom properties:

NameData TypeDefault ValueDescription
ClientModeInteger33 = Historical Alarms; 4 = Historical Events; 5 = Historical Alarms and Events
GroupByHeaderBooleanFalseWhen True, enable group by header for the alarm grid
ShowCommandBarBooleanFalseWhen True, display the command bar
Edit Custom Properties dialog showing the three custom properties: ClientMode (Integer, 3), GroupByHeader (Boolean, False), and ShowCommandBar (Boolean, False) with descriptions

Add UI Elements to the Symbol

Add a Text element with the caption "Client Mode:" and set its Element Style to Title. Add a Combo Box element (ComboBox1) to the right of the text. Double-click ComboBox1 to configure its animations: set the Selected Item Value reference to ClientMode and create three caption/value pairs — Historical Alarms (3), Historical Events (4), Historical Alarms & Events (5).

Drawing canvas with "Client Mode:" title text (Text1) and a Combo Box dropdown (ComboBox1) placed side by side
Edit Animations dialog for ComboBox1 — Static state with Selected Item Value reference set to ClientMode, and three Static Values: Historical Alarms (3), Historical Events (4), Historical Alarms & Events (5)

Add two Check Box elements: CheckBox1 labeled "Group by Header" (bound to GroupByHeader) and CheckBox2 labeled "Show / Hide Command Bar" (bound to ShowCommandBar). Align all elements using the Align Middle toolbar button. Save and close the symbol.

CheckBox1 on the canvas with "Group by Header" caption; Edit Animations dialog showing the Checked value bound to the GroupByHeader custom property
CheckBox2 labeled "Show / Hide Command Bar" on the canvas; Edit Animations dialog showing the Checked value bound to the ShowCommandBar custom property
Three aligned elements on the canvas — Client Mode label/dropdown, Group by Header checkbox, and Show/Hide Command Bar checkbox — aligned using the toolbar's Align Middle function

Update the HistoricalAlarmsEvents Layout

Open HistoricalAlarmsEvents in the Layout Editor. With Pane1 selected, add a new pane on top (Pane2). Set Pane2 properties: Height = 40, Height Sizing = Fixed, Content Alignment = Left. Drag Hist_AlarmAppConfigurations from the Toolbox to Pane2.

Pane2 properties showing Fixed height of 40 and Left content alignment; preview shows the configuration controls (Client Mode dropdown, checkboxes) above the alarm grid

With Pane1 selected, bind the following AlarmControl properties using dynamic binding (mode "In"):

PropertyBinding ModeReference
Run-Time Behavior > Group by HeaderInMyContent.Hist_AlarmAppConfigurations1.GroupByHeader
Command Bar > Show Command BarInMyContent.Hist_AlarmAppConfigurations1.ShowCommandBar
Alarm Mode > Client ModeInMyContent.Hist_AlarmAppConfigurations1.ClientMode
Properties panel showing Group By Header, Show Command Bar, and Client Mode properties all bound to the Hist_AlarmAppConfigurations symbol via dynamic references

Associate HistoricalAlarmsEvents with a Navigation Item

In the ViewApp Editor, add a new child navigation item under Home and rename it to HistoricalAlarmsEvents. Change its Title to "Historical Alarms and/or Events". Drag the HistoricalAlarmsEvents layout from the Toolbox to Pane1.

New navigation item "HistoricalAlarmsEvents" created under Home; Title property set to "Historical Alarms and/or Events"
Actions panel showing the layout assignment; preview displays Historical Alarms and/or Events with alarm data grid showing Severity, EventTime, Comment, Provider, User, and Source columns

Test the ViewApp in a Preview Session

Click Update. The preview session opens with Historical Alarms and/or Events displayed. Notice the column headings with filter icons and the Requery button at the bottom of the status bar.

Historical Alarms display with filter icons on column headers; the Severity column filter is being used to select only Severity 1 alarms
Filtered view showing Severity 1 alarms only; Requery button visible at bottom right — clicking Requery refreshes the last 15 minutes of data and clears active filters

Group by Header

Check the Group by Header checkbox at the top of the display. A group-by field appears with the prompt "Drag a column header here to group by that column." Drag the EventTime column to the group-by field to organize alarms by event time.

Group by Header enabled with EventTime column dragged to the group-by field — alarms are grouped by event time with expandable group headers showing item counts

Command Bar

Check Show / Hide Command Bar to reveal the command bar with Navigate, Requery, Connect, and Disconnect buttons. Click Requery to refresh the alarm data.

Command bar visible with Navigate, Requery, Connect, and Disconnect buttons — the Requery button is circled in red for emphasis

Historical Events

From the Client Mode combo box, select Historical Events. You may not have events for the last 15 minutes, so generate some by navigating to Mixer100, opening and closing valves, starting and stopping the agitator, and changing the speed setpoint. Then navigate back to Historical Alarms and/or Events and select Historical Events to see the generated events.

Historical Events display showing recent operational events sorted by Event Time in descending order — events include alarm activations, valve operations, and setpoint changes

Key Takeaways

  • Historical alarm displays connect to History Blocks or SQL Server databases via the Configuration Editor
  • Custom symbols with bound properties allow runtime control of Client Mode, grouping, and command bar visibility
  • The Requery button refreshes historical data for the configured time range
  • Column filter icons allow quick filtering at runtime (e.g., by severity)
  • The Group by Header feature enables drag-and-drop grouping for data organization

Knowledge Check

Q1: What Client Mode values correspond to historical alarm data?

Client Mode 3 = Historical Alarms, 4 = Historical Events, 5 = Historical Alarms and Events.

Q2: Why is "Save User Queries and Filters" unchecked in this lab?

To prevent runtime-created queries and filters from being saved permanently, ensuring the alarm display returns to its design-time configuration each session.

Q3: What does the Requery button do?

It refreshes the historical alarm/event data for the configured time range and also clears any active column filters, returning to the complete dataset.

Q4: How do you dynamically bind AlarmControl properties to a custom symbol?

Set the property's binding mode to "In" (dynamic) and enter a reference path like MyContent.SymbolName.PropertyName in the value field.
