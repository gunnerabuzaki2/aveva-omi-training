🔬 L21: Lab 10 – Enhancing the ViewApp Layout

AVEVA™ Operations Management Interface 2023 — Module 4: ViewApp Application Customizations

Pages 309–326 (Lab 10, pp. 4-23 to 4-40)

Lab Objectives
Upon completion of this lab, you will be able to:
  • Configure additional layout properties
  • Configure additional pane properties
  • Enable pan and zoom for panes
  • Configure pane dividers in a layout
  • Use the HamburgerApp

Introduction

In this lab, you will enhance the layout of the ViewApp in a number of ways. You will add and configure a pane for a hamburger button at the top of the ViewApp, and assign the HamburgerApp to it. The icon for the hamburger button is three horizontal lines — clicking it opens the slide-in pane on the left side of the ViewApp for navigation at runtime.

You will update the slide-in pane on the left of the layout so that when open, it will not hide the title bar or the hamburger button at the top of the display. You will enable the zoom and pan feature for a pane to allow zooming and panning content in the pane at runtime. You will configure pane dividers to be interactive so they can be moved at runtime. Optional steps are also provided for adding a background image to the ViewApp.

Lab 10 introduction — the AVEVA OMI Training Application with breadcrumb navigation showing the plant hierarchy from Assets down to Mixer100, with color-coded status counts at each level

Add Panes for the HamburgerApp

In the following steps, you will edit the Main layout in the Layout Editor. You will add a pane for the HamburgerButton at the top of the layout, and configure it so that its width will not change if the layout size changes in the running ViewApp.

Step 1. On the Graphics tab, in the Training \ Layouts toolset, double-click Main to open it in the Layout Editor.

Step 2. In the layout preview area, select Pane2, which has NavBreadcrumbControl assigned.

Step 3. Right-click Pane2 and select Add Pane → Left.

Right-clicking Pane2 in the Layout Editor reveals the context menu with Add Pane options — selecting "Left" creates a new pane positioned between the hamburger area and the breadcrumb

A new pane named Pane5 is added to the left of Pane2.

After adding Pane5 to the left of Pane2 — the new pane sits alongside the breadcrumb navigation, ready to host the hamburger button

Step 4. Select Pane5, and on the Properties tab, configure properties as follows:

PropertyValue
Size → Width50
Size → Width SizingFixed
Note: The Size → Height Sizing property for Pane5 is already set to Fixed because this pane was added based on Pane2, which has a fixed height.

Pane5 is resized in the preview. The following shows Pane5 selected with a width of 50 px.

Pane5 properties — a narrow 50px fixed-width pane that will hold the hamburger button, with fixed height inherited from Pane2
The layout preview after configuring Pane5 — the narrow pane is visible to the left of the breadcrumb navigation area

Step 5. On the Toolbox tab, enter a search string to find the HamburgerButton and select it.

Step 6. Drag the HamburgerButton to Pane5.

Searching the Toolbox for "hamb" reveals the HamburgerButton component — its three-line icon is the standard hamburger menu symbol used for slide-in navigation
After dragging the HamburgerButton into Pane5 — the button is now assigned and will appear at runtime as a three-line icon
Key Concept: The HamburgerApp
The HamburgerButton is a built-in OMI control that, when placed in a pane, provides a three-line menu icon at runtime. By default, clicking it opens a slide-in pane on the left side of the ViewApp for hierarchical asset navigation. The HamburgerButton's default configuration is typically sufficient — no additional setup is required for basic slide-in behavior.
Tip: Because Pane5 is very small (50×50 px), you may want to zoom the layout preview display to make it easier to assign content to the pane. Use the "Zoom to Fit" option when you want to reset the display.

Customize the Left Slide-In Pane

In the following steps, you will add a new pane to the top of the left slide-in pane. You will customize the new pane to have a transparent background, so that when the slide-in pane is open, the TitleBarControl and HamburgerButton will still be visible.

Step 7. Select Pane3 and add a new pane above it. A new pane named Pane6 is added above Pane3.

Adding Pane6 above Pane3 in the slide-in pane structure — this new top pane will become a transparent overlay so the title bar remains visible
The left slide-in pane now has Pane6 (top) and Pane3 (bottom) — Pane6 will be made transparent to serve as an overlay

Step 8. Select Pane6, and on the Properties tab, configure properties as follows:

PropertyValue
Size → Height131 (same as the height of Pane2 plus Pane4 and a divider between the panes)
Size → Height SizingFixed
Size → Hide Row When Emptyunchecked

Pane6 resizes in the preview area. Notice that the bottom line of Pane6 is aligned with the bottom lines of Pane5 and Pane2.

Pane6 properties — the fixed height of 131 pixels matches the combined height of Pane2, Pane4, and the divider between them, ensuring proper alignment
Pane6's bottom edge aligns precisely with the bottom of Pane2 and Pane5 — red arrows highlight this critical alignment that ensures the transparent overlay covers exactly the right area

Set Transparent Background on Pane6

Step 9. With Pane6 selected, on the Properties tab, click the down-arrow next to the Appearance → Background property to display the color picker.

Notice that the color picker shows RGBA color code values, where R is red, G is green, B is blue, and A is alpha. Alpha controls the transparency or opacity of the other three colors.

The color picker shows RGBA values — the Alpha channel at 100% means fully opaque. Setting it to 0% will make the background fully transparent

Step 10. Change the Alpha field on the right to a value of 0. This will make the pane background transparent.

Note: You can also change the first two digits of the color code to 00 (zero) to set the background to transparent. You may need to click off the color picker to close it.

The Appearance → Background property for Pane6 is listed as #00E4E4E4.

Setting Alpha to 0 makes Pane6's background fully transparent — the hex code updates to #00E4E4E4 where the leading "00" indicates zero opacity
Pane6's complete Appearance properties after transparency — the background is transparent (#00E4E4E4) while the pane itself remains fully visible (Opacity: 1)
Key Concept: RGBA Transparency in Pane Backgrounds
The background color uses 8-character RGBA hex codes in the format #AARRGGBB, where the first two characters represent the Alpha (transparency) channel. Setting Alpha to 00 makes the pane fully transparent. This is essential for overlay panes that should not obscure content behind them — such as the top section of a slide-in pane, which must let the title bar and hamburger button show through.

Add Padding to a Pane

Now you will configure padding for a pane, which will add a small number of blank-space pixels between the pane border and content within the pane.

Step 11. In the preview area, select Pane1.

Step 12. On the Properties tab, change the Appearance → Padding property to 5,5,5,5.

Step 13. Save and close the Main layout, and check it in.

Adding 5 pixels of padding on all sides of Pane1 creates breathing room between the pane border and its content — the four values represent top, right, bottom, and left padding
Key Concept: Pane Padding
Padding creates a transparent buffer zone between a pane's border and its content. The format 5,5,5,5 means 5 pixels of padding on the top, right, bottom, and left sides. This prevents content from touching the pane edges and improves visual readability. Padding values can be set independently for each side (e.g., 10,5,10,5 for more horizontal padding).

Enable Pan and Zoom on a Pane

Next, you will customize panes in Asset_Layout. You will enable the pan and zoom controls and configure the divider between panes to be movable at runtime.

Step 14. On the Graphics tab, in the Training \ Layouts toolset, double-click Asset_Layout to open it in the Layout Editor.

Step 15. Ensure Pane1 is selected, and configure its Appearance → Interaction Mode property to PanAndZoom.

Enabling PanAndZoom interaction mode on Pane1 — this adds zoom controls and pan interaction to the process diagram pane at runtime
Key Concept: PanAndZoom Interaction Mode
Setting a pane's Interaction Mode to PanAndZoom enables runtime zoom and pan controls at the bottom of the pane. Operators can zoom in to inspect process details and pan across large diagrams. The zoom level can be set via a dropdown (up to 5000%), and Pan Mode allows click-and-drag navigation. A "Zoom To 100%" button provides a quick reset.

Make Dividers Interactive

Step 16. Click the pane divider between Pane1 and Pane2, but do not move the divider.

Note: Do not change the position of the divider — just click it to select it and display its properties on the Properties tab.

Step 17. On the Properties tab, check the General → Is Interactive property.

Step 18. Save and close Asset_Layout, and check it in.

Selecting the divider between Pane1 and Pane2 — click carefully without dragging to select it and access its properties
The divider properties panel — enabling "Is Interactive" allows operators to drag the divider at runtime to resize adjacent panes
Key Concept: Interactive Pane Dividers
By default, pane dividers are static — they cannot be moved at runtime. Checking the Is Interactive property on a divider enables operators to drag it to resize adjacent panes. This gives end users flexibility to allocate more space to the content they need most. The divider's thickness, color, and visibility can also be customized.

What You Accomplished

Lab Summary
In this lab, you enhanced the ViewApp layout by:
  • Adding a hamburger button for slide-in navigation via the HamburgerApp control
  • Creating a transparent overlay pane (Pane6) so the title bar remains visible when the slide-in pane is open
  • Adding 5 pixels of padding to Pane1 for visual breathing room around content
  • Enabling PanAndZoom interaction mode on process diagram panes for runtime zoom and pan
  • Making pane dividers interactive so operators can resize panes at runtime
  • Optionally adding a tiled background image to the Main layout
These customizations transform a basic ViewApp into a polished, operator-friendly interface with navigation, interactive controls, and visual polish.

Knowledge Check

Q1: What is the purpose of the transparent Pane6 above the slide-in pane?

Pane6 is configured with a transparent background (Alpha = 0) so that when the slide-in pane opens, it does not obscure the title bar or hamburger button. The transparency allows content behind it — specifically the TitleBarControl and the HamburgerButton in Pane5 — to remain visible even though Pane6 physically covers that area of the layout.

Q2: How do you enable pan and zoom on a pane?

Select the pane in the Layout Editor, go to the Properties tab, and set the Appearance → Interaction Mode property to PanAndZoom. This adds a zoom level dropdown, zoom in/out buttons, and a Pan Mode toggle at the bottom of the pane at runtime. Operators can zoom up to 5000% and use Pan Mode for click-and-drag navigation.

Q3: What does the "Is Interactive" property do for a pane divider?

When checked, the divider can be dragged by operators at runtime to resize the adjacent panes. By default, dividers are static and cannot be moved. Enabling Is Interactive gives end users the flexibility to customize the layout on-the-fly, allocating more space to the content they need most.

Q4: Why doesn't the tile background effect show in the Layout Editor?

The Layout Editor does not render the tile effect — it only shows the background image once in its default position. To verify the tiled appearance, you must use the ViewApp preview session at runtime, where the image repeats across the entire layout background as configured.

Q5: What is the significance of the 131-pixel height for Pane6?

The 131-pixel height for Pane6 matches the combined height of Pane2 (the breadcrumb navigation), Pane4 (the title bar), and the divider between them. This ensures that when the slide-in pane opens, the transparent overlay covers exactly the area occupied by the title bar and breadcrumb, without extending into the main content area below.
