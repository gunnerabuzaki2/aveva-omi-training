Section 4 – Tools and Animations

Introduction

You can use tools from the Graphic Editor to draw graphic elements on the drawing canvas.

Graphic elements can be directly edited on the canvas using tools provided in the Graphic Editor,

such as menu options, toolbar options, properties, and more. Animations can also be added to the

graphic elements.

Tools Pane

The Graphic Editor provides a set of element tools that you can use to draw and select elements

on the drawing canvas.

The Tools pane is a collection of elements that you can use when creating and editing symbols.

The Tools pane includes:



Basic shapes, such as lines, rectangles, polygons, and arcs



A pointer tool to select and move elements on the drawing canvas



Common Windows controls, such as combo boxes, calendar controls, radio button

groups, and more



A Status element to indicate quality and status conditions of attributes contained in

selected animated elements and/or specified attributes

This section describes element tools and menus available in the Graphic Editor, symbol and

graphic element properties, and visualization and interaction animations.

Elements List

The Elements pane shows a hierarchical list of graphic elements and embedded symbols by name

on the drawing canvas.

The order of the items in the Elements list determines which element or embedded symbol

appears on top if they overlap on the drawing canvas. When adding a new item to the canvas, by

default, it is listed at the top of the Elements list. The order can be manually changed.

Elements and embedded symbols can be selected on the canvas or from the Elements list.

Selecting from the Elements list can be particularly useful for selecting one or more elements and

embedded symbols that are visually hidden by other elements on the canvas.

Elements and embedded symbols can be managed through the Elements list as well, such as

duplicating items, changing the order of selected items, renaming items, grouping items, and

editing items within a group.

Elements and embedded symbols can be used as references for animations and scripts within the

symbol. They are referenced in a format as .. If an element, a

group, or an embedded symbol is referenced in an animation or a script, then renaming the

element, group, or embedded symbol will not update the references automatically. Animations or

scripts referencing the old name will need to be updated manually.

Edit Operations

Editing operations assist you in editing elements and symbols on the drawing canvas. Most of the

operations are available from the Edit menu and from toolbar buttons. Some of the operations

include:



Undoing and redoing operations



Duplicating selected elements



Cutting, copying, and pasting elements



Deleting elements



Selecting all elements on the canvas

Arrangement Operations

Arrangement operations allow you to arrange elements and symbols on the drawing canvas. Most

of the operations are available from the Arrange menu and from toolbar buttons. Some of the

operations include:



Sending elements backward or forward



Aligning elements to the left, right, top, or bottom



Rotating or flipping elements



Distributing elements horizontally or vertically



Making elements the same size



Grouping multiple elements or ungrouping them



Creating a path graphic from multiple open line elements

Format Operations

Formatting operations allow you to specify formats for text and lines on the canvas. Most of the

operations are available from the Format menu and from toolbar buttons. Some of the operations

include:



Specifying font settings for text



Aligning text



Specifying line settings, such as weight, pattern, ends, and more



Specifying transparency

Graphic Element Properties

When a graphic element is selected on the drawing canvas, its properties appear on the

Properties tab. The properties are organized into categories such as the ones listed below, as

appropriate for the graphic element.



Graphic: Name of the selected graphic element.



Appearance: Properties related to the appearance of the selected graphic element, such

as X and Y location, size, orientation, element style, and more.



Fill Style: Properties related to the fill of the selected graphic element, such as fill

orientation, color, direction, and more.



Line Style: Properties related to the line appearance of the selected graphic element,

such as line weight and line color.



Text Style: Properties related to text appearance of the selected graphic element.



Runtime Behavior: Properties related to visibility, tab order, and other runtime behaviors.

Different types of graphic elements may have different properties. For example, a text element has

text style properties, such as font and text color, but it does not have fill or line style properties. If

multiple graphic elements are selected, only their common properties are listed on the Properties

tab.

ElementStyle Property

One of the properties in the Appearance category is the ElementStyle property, which you can use

to apply a style to elements on the drawing canvas. Using the ElementStyle property is a way to

standardize the appearance of elements in symbols in a Galaxy. Applying styles to graphic

elements overrides the native visual properties of the graphic elements, such as font, fill color, line

color, line weight, and transparency.

To apply the ElementStyle property to one or more elements on the drawing canvas, select them

directly on the canvas or from the Elements list. Then, on the Properties tab, select the

ElementStyle property to show the drop-down arrow for the property. Click the drop-down arrow for

the ElementStyle property to view a list of styles.

Predefined styles are provided, as well as a set of user-defined styles. The available styles are

organized in categories, such as HMI, Alarm, and Trend. You can scroll to find the style you want

or use the Search field at the top to search for it.

The available styles in the drop-down list for the ElementStyle property and their configurations

are based on the active style in the Galaxy. Refer to Section 6, “Galaxy Styles” for more

information about Styles.

Animations

Animations can be used to change the appearance of graphic elements and allow users to interact

with the view application at runtime. Some animations can be driven by data from automation

objects, element properties, and other data sources.

There are two types of animations:



Visualization animations determine the appearance of the element, such as blinking, fill

style, percent fill horizontal, and value display.



Interaction animations determine the behavior of an element, such as horizontal sliders

and user input.

There are some animations that are specific to certain elements. For example, the DataStatus

animation is specific to the Status element. Element-specific animations also determine element

behavior and appearance. Additionally, some types of elements do not support certain animations.

For example, a rectangle element that does not contain text will not support the Value Display

animation that requires text.
