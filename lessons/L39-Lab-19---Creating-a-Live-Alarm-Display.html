📖 L39: Lab 19 – Creating a Live Alarm Display and Managing Alarms at Runtime

Module 9 — Alarms and Events Visualization, Section 2

Pages 543–570

Objectives

Upon completion of this lab, you will be able to:

  • Configure the AlarmApp to display current alarms
  • Create default alarm queries for the AlarmApp
  • Create and manage additional queries and filters
  • Customize appearance and behavior configurations
  • Acknowledge, shelve, and unshelve alarms at runtime
  • Hide and unhide alarms at runtime

Content workflow highlight: Configuring a pane's presentation style to allow support of multiple content items.

Create a Layout and Assign the AlarmApp

In this first section, you create a new layout named LiveAlarms and assign the AlarmControl to it. On the Graphics tab, in the Training > Layouts toolset, create a new layout and name it LiveAlarms. Double-click LiveAlarms to open it in the Layout Editor. On the Toolbox tab, search for the AlarmControl and drag it to Pane1.

The LiveAlarms layout in the Layout Editor with the AlarmControl widget placed in Pane1 via drag-and-drop from the Toolbox tab

Configure the AlarmApp

On the Properties tab, click the ellipsis button to the right of the Configuration property to open the Configuration dialog box.

Properties tab showing General section (AlarmControl1, AlarmApp.AlarmControl, Pane1), Run-Time Behavior checkboxes, and the Configuration ellipsis button (red arrow)

Set the Alarm Query

On the Alarm Mode tab, change the default Alarm Query to point to your production node's Galaxy Plant area. The format is \\<ProductionNodeName>\Galaxy!Plant. Your instructor will provide the production node name — for example, \\S00PROD\Galaxy!Plant.

Configuration Dialog — Alarm Mode tab with Client Mode set to "Current Alarms" and the Alarm Query field populated with the production node query path

Configure Column Details

Select the Column Details tab and rename User1 to Severity, and Group to Area. Check the Provider checkbox. The preview at the bottom of the tab updates to reflect the display name changes.

Column Details tab showing the list of alarm fields with their display names, widths, and original names — Severity, Area, and Provider are highlighted with red arrows
Column Details tab with preview pane showing renamed headers; Sorting configured with TimeLCT Descending as the first sort criteria

Import Queries and Filters

Select the Queries and Filters tab. Note that the Default query does not yet reflect the query from the Alarm Mode tab — switch to Alarm Mode and back to sync it. Then click Import to load a prebuilt XML file with alarm queries and filters from C:\Training.

Queries and Filters tab showing the Default query before and after synchronization with the Alarm Mode tab; the Import button is highlighted for loading external query files
Important: When importing, click No on the overwrite prompt to avoid replacing the existing Default query. After import, verify that the "Shelved Alarms" query and "Critical", "High", "Medium", and "Low" filters appear in the list.
Open dialog navigating to the Lab 19 prebuilt XML file, followed by the Import Query and Filter overwrite prompt and success confirmation
Queries and Filters tab after import — showing Default, Shelved Alarms queries, and Critical/High/Medium/Low severity filters

Enable Area Hierarchy and Disable Command Bar

On the Properties tab, uncheck Show Command Bar and check Show Area Hierarchy. Save and close the LiveAlarms layout, and check it in.

Property panel with Command Bar section (Show Command Bar unchecked, red arrow) and Area Hierarchy section (Show Area Hierarchy checked, red arrow)

Update the Main Layout with a Tabbed Pane

Open the Main layout in the Layout Editor. Select Pane7 at the bottom and add a new pane (Pane8) to the left. Resize Pane7's width to 300 pixels. Then select Pane8 and change its Presentation Style to Tabbed using the Change Presentation Style button.

Layout Editor showing the new Pane8 added to the left of Pane7; Pane7 properties show Content Type: Alarm Summary and Width: 300
The Change Presentation Style dropdown showing Single, Multiple, and Tabbed options — select Tabbed for Pane8

Notice the icon at the top of Pane8 changes to indicate it is a tabbed pane. Save and close the Main layout.

Pane8 with the tabbed pane icon visible at the top, ready for content assignment in the ViewApp Editor

Associate LiveAlarms with a ViewApp Navigation Item

Switch to the ViewApp Editor. In the Navigation list, select Plant. On the Toolbox tab, search for the LiveAlarms layout and drag it to Pane8. Because the content is assigned to a tabbed pane, it does not display in the preview area — verify the assignment in the Actions list.

Navigation panel with Plant selected, and Actions panel showing "Show: LiveAlarms From: Plant On: Pane8" (highlighted in blue)

Test the ViewApp in a Preview Session

Click Update to update the preview session. Navigate to Plant. The LiveAlarms tab should appear with alarms listed in the alarm pane, and Plant.AlarmSummary in the adjacent pane.

AVEVA OMI Training Application preview — Simulated Factory Layout with Receiving, Production, Packaging, Shipping areas; LiveAlarms grid showing Severity, State, Node, Area, Name, AlarmComment columns; Alarm Summary showing Critical:2, High:25, Medium:12

Context-Sensitive Alarm Display

Navigate to Production Lines — the AlarmApp is configured to follow the context, so all alarms from the production area (including subareas Line1 and Line2) are displayed. Navigate to Line1 — only Line1 alarms appear.

Navigating to Production Lines shows all subarea alarms; navigating to Line1 filters to only Line1 alarms — demonstrating the Follow Current Context behavior

Area Hierarchy Independent Navigation

Navigate to Packaging in the ViewApp. Then use the Area Hierarchy in the AlarmApp to navigate to Line1. The Area Hierarchy sets the alarm query independently — it does not affect the current navigation in the ViewApp, so the ViewApp still shows Packaging.

AlarmApp's Area Hierarchy independently navigated to Line1 while the ViewApp remains on Packaging — demonstrating independent alarm query control

Query and Filter at Runtime

Right-click the alarm grid and select Queries and Filters. Enable the Critical filter and click OK. Only critical alarms (Severity 1) appear. Double-click a mixer alarm to navigate to that equipment in the ViewApp.

Query and Filter Favorites dialog with the Default query and Critical/High/Medium/Low filters — Critical is checked to filter by critical severity
LiveAlarms with Critical filter applied showing 13 alarms; double-clicking a mixer alarm navigates the ViewApp to Mixer300 — note the Area Hierarchy does not follow the double-click navigation

Acknowledge Alarms

Switch to the Medium filter. Select an alarm with state UNACK, right-click and choose Ack Selected. Enter a comment such as "Ticket has been issued" and click OK. The alarm state updates to ACK with your comment visible in the AlarmComment column.

Right-click context menu with Ack Selected; Ack Comment dialog with comment text; updated grid showing ACK state with "Std ACK - Ticket has been issued" comment

Then right-click without selecting a row and choose Ack Others | Ack All. Enter "Ack all alarms" as the comment. All active alarms display with ACK state.

Ack All operation (left) — Ack Comment dialog with "Ack all alarms"; Shelve All operation (right) — Shelve Comment dialog with 1-hour duration and reason text

Shelve and Unshelve Alarms

Shelve all alarms by right-clicking and selecting Shelve Others | Shelve All. Enter a reason and duration. All alarms disappear. To view shelved alarms, open Queries and Filters and enable the Shelved Alarms query. Shelved alarms reappear with SHELVE_UNAC state. Then select Unshelve Others | Unshelve All to restore them.

Query and Filter Favorites with Shelved Alarms query checked; shelved alarms displayed in the grid showing SHELVE_UNAC state
Unshelve All operation restoring alarms to the display; query reset to Default with Medium filter active

Freeze, Hide, and Unhide

Clear all filters, then right-click and select Freeze to pause alarm updates. A freeze icon appears on the status bar. Select Freeze again to unfreeze. To hide alarms, right-click and select Hide Others | Hide All — all alarms disappear and a hidden count appears in the status bar tooltip. Open the Hidden Alarms dialog, select all alarms, and click Unhide to restore them.

Frozen alarm grid (26 alarms displayed, freeze icon on status bar); unfreezing restores live updates
Empty alarm grid after Hide All (Hidden Alarms=26 tooltip); Hidden Alarms dialog showing the table of hidden alarms with Item, Node, Provider, Group, Priority, Name, Class columns
All 26 alarms restored to the alarm display after unhide, with Area Hierarchy tree visible (Plant → Production Lines → Line1 → Line2)

Key Takeaways

  • The AlarmApp integrates seamlessly with navigation context via the Follow Current Context property
  • The Area Hierarchy provides independent navigation within the alarm display, separate from ViewApp navigation
  • Queries and Filters can be configured at design time and managed at runtime
  • Runtime alarm management includes acknowledge, shelve/unshelve, and hide/unhide operations
  • The Tabbed presentation style allows multiple content items in a single pane

Knowledge Check

Q1: What layout presentation style allows multiple content items in a pane?

The Tabbed presentation style. Change it using the Change Presentation Style button at the top of the pane in the Layout Editor.

Q2: What happens when you use the Area Hierarchy in the AlarmApp to navigate to a different area?

The Area Hierarchy sets the alarm query independently. It does not change the current navigation in the ViewApp — the ViewApp remains on its current page while alarms for the newly selected area are displayed.

Q3: How do you view shelved alarms?

Open the Query and Filter Favorites dialog (right-click the alarm grid) and enable the "Shelved Alarms" query. Shelved alarms then appear in the grid with SHELVE_UNAC state.

Q4: What is the difference between hiding and shelving alarms?

Hiding alarms removes them from the grid display but does not affect the actual alarm generation — records are still logged to history. Shelving alarms temporarily suppresses them for a configurable duration with a mandatory comment and reason.
