🔬 L11: Lab 5 – Creating Symbols for Automation Objects

AVEVA™ Operations Management Interface 2023 — Module 3, Lab 5

Pages 187–206 (Section 3-49 to 3-68)

Lab Introduction

In this hands-on lab, you will create visual symbols for equipment modeled in the Galaxy — such as valves, agitators, pumps, level meters, and temperature sensors. You will use pre-built symbols from the Situational Awareness Library and configure them to match the specific requirements of each piece of equipment.

Once the new symbols are created and configured, you will associate them with corresponding template objects using the Link Content feature. This is the critical step that connects your visual symbols to the automation objects in your Galaxy, enabling the ViewApp to display the right symbol at the right level of the navigation hierarchy.

Lab Objectives
Upon completion of this lab, you will be able to:
  • Link symbols to template objects using the Content pane
  • Configure symbols with custom properties using relative references
  • Duplicate symbols to quickly create variations for different equipment types

Quiz

Question 1

Why do you set the ContentType property to "Detailed" when creating symbols for individual equipment objects?

Click to reveal answer

The ContentType property determines when the symbol is displayed in the ViewApp. Setting it to "Detailed" means the symbol will be shown when the user navigates deep enough in the hierarchy to reach an individual equipment object. Pane1 in the Main layout is configured for both "Overview" and "Detailed" content types, so it automatically switches between the Overview symbol (at higher hierarchy levels) and the Detailed symbol (at the equipment level).

Question 2

What is the purpose of the Substitute References feature when configuring the agitator symbol?

Click to reveal answer

The Substitute References feature allows you to find and replace reference strings in bulk. For the agitator, it was used to replace Me.PV with Me.Speed.PV and Me.SP with Me.Speed.SP, redirecting the symbol's data references from the top-level PV/SP attributes to the Speed sub-attributes. This is necessary because the agitator's primary process variable is its speed, not a direct PV attribute.

Question 3

What does the "Use For Auto-Fill" checkbox do when configuring a pane's Content Type(s)?

Click to reveal answer

When "Use For Auto-Fill" is enabled, the ViewApp automatically fills the pane with the appropriate symbol based on the current navigation context. If the user is at a high hierarchy level (like Line1), the Overview symbol is displayed. As the user navigates deeper to individual equipment (like Agitator_001), the system automatically switches to the Detailed symbol. This eliminates the need for manual content switching.

Question 4

Why is it important to check in template objects after linking symbols, even though it produces a propagation warning?

Click to reveal answer

Checking in a template object propagates the changes (including the linked symbols) to all derived objects. The warning about propagating to multiple derived objects is expected — it means every instance of that template will now have access to the linked symbol. Without checking in, the changes remain uncommitted and won't be visible to other users or at runtime.

Key Takeaways

  1. Always set ContentType to "Detailed" for symbols representing individual equipment.
  2. The Situational Awareness Library provides pre-built symbols (SA_Meters, SA_Valve_And_Damper, SA_Agitator_Settler, SA_Pump_Blower_RotaryValve) with configurable Wizard Options.
  3. Duplicate existing symbols to quickly create variations — change only the Wizard Options that differ.
  4. Use Substitute References to bulk-update reference paths (e.g., Me.PV → Me.Speed.PV).
  5. Link symbols to template objects via the Content pane using the Link Content button.
  6. Configure panes with both Overview and Detailed content types and enable Use For Auto-Fill for automatic symbol switching.
