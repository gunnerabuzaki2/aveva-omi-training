Section 5 – Custom Properties

Introduction

You can use custom properties to customize embedded symbols to extend the functionality of the

symbol. A custom property can contain:



A value



A valid expression



An attribute from the automation object



A property of an element or an embedded symbol



A custom property of an embedded symbol



A script function

If public custom properties exist for an embedded symbol, they display on the Properties tab in the

Custom Properties category when the embedded symbol is selected.

Create and Edit Custom Properties

You create and edit custom properties with the Edit Custom Properties dialog box. To manage

custom properties for the symbol you are creating, click the drawing canvas to ensure no element

or embedded symbol is selected. Then, right-click the canvas and select Custom Properties. The

Edit Custom Properties dialog box appears. To add a new custom property, click the Add custom

property button. To remove a custom property, select a custom property from the list and click

Remove custom property. To rename a custom property, select a custom property from the list and

click the name of the custom property to make it editable.

This section provides an overview of custom properties for symbols.



Data Type: Select a data type from the drop-down list. These include Boolean, Double,

ElapsedTime, Float, Integer, String, Time, and HistorySummary.



Default Value: The initial default value for the custom property. It can be a literal value,

reference, or expression. You can click the ellipsis button to browse for a reference

through the Galaxy Browser.

If the selected Data Type is String, Time, or ElapsedTime, click the button to the left of the

Default Value field to indicate whether the default value is a static value or a reference to a

value.



Static Text (

): This icon indicates the default value is a static value. This mode

treats the Default Value field as text.



Expression or Reference (

): This icon indicates that the default value is an

expression or a reference. This mode treats the Default Value field as a lookup list or

evaluation of a mathematical expression or logical expression.



Visibility: Determines whether the custom property is visible to be configured when the

symbol is used in other symbols or layouts. Any property marked as Private is

inaccessible when the symbol is embedded in another symbol. This can be used to hide

configurations that you do not want designers to modify.



A Public property is visible when the symbol is embedded in other symbols or

assigned to layout panes.



A Private property is hidden when the symbol is embedded in other symbols or

assigned to layout panes.



Description: A description can provide a brief note about what the symbol can do. For

symbols from a built-in library, the description contains helpful tips on how to use the

custom property. For user-created custom properties, it is useful to provide helpful tips or

hints on how to properly use the custom property as intended.
