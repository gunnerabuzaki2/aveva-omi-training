📖 L07: Lab 3 – Creating and Running a ViewApp

Module 2 — Getting Started | Lab 3

Pages 107–138 | AVEVA™ Operations Management Interface 2023

Introduction

In this lab you will perform the steps necessary to create a ViewApp. You will derive a template from the base $ViewApp template, initialize it with the screen profile and layout you created in the previous lab, review the derived template in the ViewApp Editor, and test it with a preview session. You will then create an instance from the derived template, deploy it to the production node, and launch the ViewApp in runtime.

After the initial deployment, you will switch back to the engineering node, modify the Main layout by adding a title bar, and redeploy the ViewApp. On the production node you will observe the live-update behavior when the running ViewApp receives new configuration.

Lab Objectives: Upon completion you will be able to create a ViewApp template, initialize it with a screen profile and layout, use preview mode, deploy a ViewApp instance to runtime, and deploy changes to a running ViewApp.

Creating a ViewApp Template

On the Templates tab, expand TrainingGalaxy, right-click the Training toolset, and select New Template Toolset. A new toolset with the default name is created under Training.

Right-clicking the Training toolset to create a new template toolset for organizing ViewApps

Rename the new toolset to ViewApps. Then expand the TrainingGalaxy \ System toolset to reveal the base system objects, including $ViewApp.

The renamed ViewApps toolset and expanded System folder showing base templates including $ViewApp, $ViewEngine, and $WinPlatform

Right-click the $ViewApp object and select New > Derived Template to create a new derived ViewApp template.

Right-click context menu on $ViewApp showing the New > Derived Template option highlighted
Tip: A derived template is created with a default name in the System toolset. You can then rename it and move it to your custom toolset.

A derived template appears with the default name $ViewApp_001 in the System toolset. Rename it to $ViewApp_Training and move it to the Training > ViewApps toolset. Notice the error icon — it indicates the template has not yet been configured.

$ViewApp_Training renamed and moved to the ViewApps toolset, with an error icon indicating it still needs configuration

Initializing the ViewApp

Double-click $ViewApp_Training to open it in the ViewApp Editor. The ViewApp Initialization wizard appears. The first step is to select a screen profile — both the DualScreen and SingleScreen profiles you created previously are listed.

Select the SingleScreen screen profile and click Next.

The ViewApp Initialization wizard step for selecting a screen profile, showing both DualScreen and SingleScreen options

The next step, Associate Layout, appears. The Main layout you created in the previous lab is listed and already selected for the Primary screen. A preview of the screen with the layout appears on the left. Click Finish.

The Associate Layout step of ViewApp Initialization, with the Main layout pre-selected for the Primary screen
The ViewApp Editor opens after initialization, showing the navigation tree, actions list, screen profile, and layout preview

Preview and Test Navigation

The ViewApp Editor now reflects your selections: the SingleScreen profile is active, the Main layout is displayed, and the Actions panel shows auto-fill content assigned to the layout panes. You can verify the navigation hierarchy and pane content assignments before testing.

In the top-right corner of the ViewApp Editor, click Preview to open a preview session.

The ViewApp Editor toolbar with the Preview button highlighted — used to test the ViewApp without deploying

A message appears indicating that preview mode is for testing only and not intended for production use. Check Do not show this message again and click OK.

The Preview Mode warning dialog explaining this mode is for iterative development, not production use
After closing the warning, the ViewApp preview session opens and appears as an icon in the Windows taskbar
The Preview Mode warning with "Do not show this message again" checkbox selected

When the preview session opens, Pane2 at the top takes up the entire display area and Pane1 does not show. This is because panes configured for AutoFill are set to Hide When Empty by default, and there is no content to fill Pane1 automatically based on the current navigation. Pane2 is configured with Height Sizing set to Variable by default, so it expands to fill the available space.

The preview session opens with Pane2 filling the entire display — Pane1 is hidden because it has no content to auto-fill
The Display Slide-In Pane button on the left side of the layout, which reveals the NavTreeControl in Pane3

Click the Display Slide-In Pane button on the left side to view Pane3, which contains the NavTreeControl. A search field at the top of the navigation tree allows you to quickly find navigation items. Enter a search string such as mix to find matching items.

The Search field at the top of the navigation tree, used to quickly find navigation items by name
Search results showing 8 navigation items matching "mix", with Mixer400 highlighted in the results list

Select a navigation item from the results list. In this example, Mixer100 is selected. The asset tree shows the full hierarchy from Assets through Plant > Production > Line1 > Mixer100, with status indicators at each level.

Mixer100 selected in the navigation tree, showing the full asset hierarchy with alarm status indicators at each level

Click off the slide-in pane or press ESC to close it. The NavBreadcrumbControl now shows the path: Assets > Plant > Production > Line1 > Mixer100. Each level has a dropdown arrow and status indicators.

The NavBreadcrumbControl at the top of the preview showing the navigation path to Mixer100 with status indicators
The Minimize button at the top-right of the preview session, used to switch back to the ViewApp Editor

Editing the Layout from ViewApp Editor

From the ViewApp Editor, you can open the Main layout for editing in the Layout Editor. Click the vertical ellipsis on the right side of the Layout field, which shows the assigned layout is Main, and select Edit Layout.

The vertical ellipsis button next to the Layout field, with the Edit Layout option selected to open the Main layout in the Layout Editor
The Main layout opens in the Layout Editor, ready for modification of pane properties

In the Layout Editor, select Pane2, right-click it, and select Pane Properties. On the Properties tab, change the Size > Height Sizing property to Fixed. This prevents Pane2 from expanding to fill the entire display at runtime.

Right-click context menu on Pane2 showing the Pane Properties option for editing sizing behavior
The Properties tab showing Pane2's Size section with Height Sizing changed to Fixed and Height set to 100

Save and close the Main layout, and check it in. Switch back to the preview session — the changes are reflected immediately. Pane2 with the NavBreadcrumbControl no longer takes up the whole screen.

After saving the layout change, the preview session shows Pane2 with fixed height — it no longer fills the entire screen
Tip: Changes made in the Layout Editor are automatically reflected in the preview session when Auto-Update is enabled, allowing rapid iterative development.

In the top-right of the preview, click Close to close the preview session.

The Close button at the top-right of the preview session to exit the preview

Creating a ViewApp Instance

Close the ViewApp Editor and check in the $ViewApp_Training template. Now you will create a ViewApp instance from the derived template. On the Templates tab, right-click $ViewApp_Training and select New > Instance.

Closing the Layout Editor, then right-clicking $ViewApp_Training to create a new instance via New > Instance
The context menu on $ViewApp_Training showing the Instance option under the New submenu

In the Deployment view, a new instance with the default name ViewApp_Training_001 is created under UnAssigned Host. Leave the default name for now.

The Deployment view showing the newly created ViewApp_Training_001 instance under UnAssigned Host

Creating a ViewEngine

To deploy the ViewApp instance to the runtime environment, it must be hosted by a ViewEngine. Create a derived template from $ViewEngine, rename it to $gViewEngine, and move it to the Training > Global toolset.

Creating $gViewEngine by deriving from the base $ViewEngine template and moving it to the Training > Global toolset

Create a new instance from $gViewEngine and rename the instance to ViewEngine1.

ViewEngine1 instance created and displayed in the Deployment view under UnAssigned Host

Deploying the ViewApp

In the Deployment view, assign ViewEngine1 to PROD_Platform1 — the production visualization client node. Then assign ViewApp_Training_001 to ViewEngine1.

Deployment tree after assigning ViewEngine1 and ViewApp_Training_001 to PROD_Platform1

In the Model view, assign both ViewEngine1 and ViewApp_Training_001 to the Sys area.

The Model view showing ViewApp_Training_001 and ViewEngine1 assigned to the Sys area alongside AppEngine1 and GRPlatform

Right-click ViewEngine1 and select Deploy. Leave the default deployment options and click Deploy. Notice the orange synchronization icon on the ViewApp instance while files are being transferred — once complete, the icon changes to the standard ViewApp icon.

Left: ViewApp_Training_001 showing the orange sync icon during file transfer. Right: The icon changes to the standard ViewApp icon when deployment completes
Important: The orange synchronization icon indicates files are still being transferred to the target node. Wait for the icon to change before attempting to launch the ViewApp.

Launching the ViewApp in Runtime

Switch to the production node and open the AVEVA OMI Application Manager. VIEWAPP_TRAINING_001 appears in the list of deployed ViewApps. Click LAUNCH to launch it.

The AVEVA OMI Application Manager on the production node showing VIEWAPP_TRAINING_001 with the LAUNCH button ready to start the ViewApp

The ViewApp opens maximized to full screen. You can use the NavBreadcrumbControl at the top or the NavTreeControl in the left slide-in pane to navigate through the application.

The running ViewApp navigated to Mixer100, with the NavBreadcrumbControl showing Assets > Plant > Production > Line1 > Mixer100

Editing the Main Layout and Deploying Changes

Leave the ViewApp running on the production node and switch back to the engineering node. On the Graphics tab, open the Main layout in the Layout Editor. Select Pane2, right-click it, and select Add Pane > Above to add a new pane named Pane4.

Right-click context menu on Pane2 with Add Pane > Above option to insert a new pane above Pane2
A new Pane4 added above Pane2 in the layout hierarchy, with Pane3 visible on the left and Pane1 at the bottom

Pane4 inherits the Fixed Height Sizing from Pane2. On the Properties tab, change the Size > Height property to 80.

Pane4's Properties tab showing inherited Fixed Height Sizing with Height initially at 49
After changing Pane4's Height to 80, with Aspect Ratio automatically updated to 24:1

On the Toolbox tab, search for TitleBarControl and drag it to Pane4.

The Toolbox tab with "title" entered in the search field, locating the TitleBarControl for use in Pane4
Dragging the TitleBarControl from the Toolbox to Pane4, which is highlighted as the drop target

Configuring the TitleBarControl

With Pane4 selected, change the Content and Display Area > Title property to AVEVA OMI Training Application. Then click outside the panes to select the layout and display layout properties on the Properties tab.

The TitleBarControl Properties tab showing the Title set to "AVEVA OMI Training Application" with font and color settings
The Layout properties panel accessible by clicking outside the panes, showing title mode and target settings

On the Properties tab, uncheck the Behavior > Has Title Bar property. This removes the default window title bar and replaces it with the TitleBarControl you just configured.

The Layout properties showing Behavior section with Has Title Bar unchecked — this replaces the default title bar with the custom TitleBarControl
After deploying the changes, switching to the production node triggers an "Open Updated ViewApp?" dialog

Save and close the Main layout, check it in, and deploy the changes. Switch to the production node — an Open Updated ViewApp message appears. Click OPEN NOW.

The updated ViewApp showing the custom TitleBarControl with "AVEVA OMI Training Application" title, replacing the default window title bar
Key Takeaway: The ViewApp supports hot-deployment — changes to layouts and configuration can be pushed to a running ViewApp without restarting. The TitleBarControl provides a customizable title bar that can be styled and configured independently of the window frame.

Quiz

Test Your Knowledge

1. What is the purpose of the ViewApp Initialization wizard?

The Initialization wizard appears the first time a derived ViewApp template is opened. It allows you to select a screen profile and associate layouts with the screens in that profile. After clicking Finish, the ViewApp Editor opens with the selected configuration.

2. Why does Pane1 remain hidden in the preview session when first opened?

Panes configured for AutoFill are set to Hide When Empty by default, and there is no content assigned to automatically fill Pane1 based on the current navigation. Pane2 expands to fill the available space because its Height Sizing is set to Variable by default.

3. What does the orange synchronization icon on the ViewApp instance indicate?

The orange synchronization icon indicates that the ViewApp files are in the process of being transferred to the target node. Once the file transfer is complete, the icon changes to the standard ViewApp icon, indicating deployment is complete.

4. What is the role of the ViewEngine in deploying a ViewApp?

The ViewEngine is required to host a ViewApp instance in the runtime environment. The ViewApp instance must be assigned to a ViewEngine, which in turn must be assigned to a platform (such as PROD_Platform1) that serves as the production visualization client node.

5. How do you replace the default window title bar with a custom TitleBarControl?

Add a new pane above the existing panes, set its Height Sizing to Fixed with a Height of 80, drag the TitleBarControl from the Toolbox to this pane, configure the Title property, and uncheck the Behavior > Has Title Bar property on the layout. This removes the default title bar and displays the custom TitleBarControl instead.
