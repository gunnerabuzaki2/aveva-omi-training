Section 2 – Custom ViewApp Namespaces and Attributes

Introduction

You can create a ViewApp namespace from the Graphics tab in the System Platform IDE. The

Graphics tab does not contain a default ViewApp namespace.

To create a ViewApp namespace, you can:



Use the New | ViewApp Namespace option from the Galaxy menu



Press Ctrl + Shift + V



Right-click in the Graphics tab, and then select New | ViewApp Namespace

Once the ViewApp namespace is created, you can double-click it in the Graphics tab to open the

ViewApp Namespace Editor.

ViewApp Namespace Editor

You use the ViewApp Namespace Editor to create and configure attributes that can be used by all

ViewApps in a Galaxy.



(1) Menu Bar: Commands to save, save and close, and close the ViewApp Namespace

Editor.



(2) Command Bar: Commands to add, duplicate, and remove attributes from the list of

attributes.



(3) Search Field: Allows you to enter text to filter the attributes in the list. The text can

appear in any part of the attribute name. The Modified Only check box filters for attributes

with unsaved modifications.



(4) Attributes List: List of attributes defined for the ViewApp namespace. The columns

ViewApp Namespace Editor.

Add Attributes

In the ViewApp Namespace Editor, you use the Add command to add a new attribute to the list of

attributes.

An attribute is added to the list with the default name of Attribute00n, but you can rename it.

Configure Attributes

Once an attribute is added to the list of attributes, you can configure the attribute:



Name: Name of the attribute.



Type: Data type of the attribute. Can be Boolean, Integer, String, Float, Double, Time, or

ElapsedTime. The default data type is Boolean.



Initial Value: The initial value for the attribute at runtime.

For Boolean attributes, a check box is provided in the Initial Value field. Checking the

check box sets the initial value for the Boolean attribute to true. The default initial value for

a Boolean attribute is false (the check box is unchecked).

Note: The Initial Value field does not support expressions.  Readonly: When checked, designates the attribute as a read-only attribute. The default value is false (not read-only, but read/write).  Retentive: When checked, designates the attribute as a retentive attribute. This means that the value of the attribute will be maintained if the ViewApp is shut down, and restored to that value when the ViewApp is restarted. However, the value of the attribute will not be retained when a ViewApp is undeployed. Note: An attribute cannot be designated as both read-only and retentive. The default value is false (non-retentive).  Description: Text that describes the attribute.

Duplicate Attributes

You can use the Duplicate command to duplicate an attribute in the list of attributes. Just select an

attribute in the list and click Duplicate.

The new attribute has all of the properties of the attribute from which it was duplicated. You can

configure the new attribute as needed.

You can select multiple attributes using the Shift or Ctrl keys and duplicate all selected attributes in

a single operation.

Remove Attributes

You can use the Remove command to remove an attribute. Just select an attribute and click

Remove.

You can select multiple attributes using the Shift or Ctrl keys and remove all selected attributes in a

single operation.

Search for Attributes

You can enter a search string to filter the list of attributes to only attributes that contain the search

string in their name. This might be useful if there are many attributes defined for the namespace.

If the “Modified Only” check box is checked, only attributes with unsaved modifications are shown.

Import and Export ViewApp Namespaces

ViewApp namespaces can be exported as standalone, reusable objects for use in other Galaxies.

For example, you might create a ViewApp namespace that is broadly applicable across a variety

of Galaxies for multiple locations, and export it as an .aaPKG file. Once you import that ViewApp

namespace into another Galaxy, you can customize it in that Galaxy as needed.

When exporting a ViewApp, all ViewApp namespaces in the Galaxy are exported with it.
